Part 0 · 导论
信息论：通信的数学基础

1948 年，Claude Shannon 发表了划时代论文 A Mathematical Theory of Communication，为一门全新的学科奠定了基础。信息论（Information Theory）的核心问题是：在不可靠的信道上，我们能可靠地传输多少信息？为了回答这个问题，Shannon 引入了三个关键概念：

  • 信息熵（Entropy）：信源产生信息的速率有多快？
  • 信道容量（Channel Capacity）：信道能可靠传输信息的最大速率是多少？
  • 编码定理（Coding Theorems）：如何设计编码方案来逼近理论极限？

Shannon 的通信系统模型将通信过程抽象为五个组成部分：

组件符号作用
信源$X$产生消息（文本、图像、语音）
编码器$f$将消息映射为适合信道传输的信号
信道$p(y|x)$传输信号，可能引入噪声
解码器$g$从接收信号恢复原始消息
信宿$\hat{X}$接收重建的消息

这套模型给出了信息论最根本的两条定理：

  • 信源编码定理（第一定理）：熵是无损压缩的理论下界
  • 信道编码定理（第二定理）：只要传输速率低于信道容量，就存在编码方案使错误概率任意小

前置知识回顾

本笔记默认你熟悉以下概念。如果某个概念不熟，可以先回去补课。

  • 概率论基础：联合分布 $p(x,y)$，条件概率 $p(x|y) = p(x,y)/p(y)$，贝叶斯定理
  • 随机变量：离散随机变量及其分布，连续随机变量的概率密度函数
  • 对数函数：$\log_2$ 运算的基本性质，换底公式 $\log_a b = \log_2 b / \log_2 a$
  • 期望：$\mathbb{E}[f(X)] = \sum_x p(x) f(x)$
Part 1 · 基础度量
自信息与信息熵

自信息：一个事件携带多少信息？

直觉上，越不可能发生的事件，一旦发生，带给我们的"信息"越多。"太阳从东方升起"几乎不包含信息，但"某只股票今天涨了 50%"就包含了大量信息。Shannon 将这种直觉形式化：

$$I(x) = -\log p(x) = \log \frac{1}{p(x)}$$

其中 $p(x)$ 是事件 $x$ 发生的概率。对数底数的选取决定了信息的单位：

底数单位适用场景
$2$比特（bit）数字通信、数据压缩
$e$奈特（nat）理论推导、与物理熵对接
$10$哈特莱（Hartley）某些工程应用
数值例子：掷一枚均匀硬币，正面概率 $p = 0.5$，自信息 $I = -\log_2 0.5 = 1$ 比特。掷一枚均匀六面骰子，某个点数概率 $p = 1/6$，自信息 $I = -\log_2(1/6) \approx 2.58$ 比特。骰子结果的不确定性更高，所以每个结果携带更多信息。

信息熵：随机变量的平均信息量

自信息度量的是单个事件的信息量。对于一个随机变量 $X$（比如一封邮件、一张图片），我们更关心它的平均不确定性，这就是信息熵：

$$H(X) = -\sum_{x \in \mathcal{X}} p(x) \log p(x) = \mathbb{E}[I(X)] = \mathbb{E}[-\log p(X)]$$

理解信息熵的几个关键性质：

  1. 非负性：$H(X) \geq 0$，等号成立当且仅当 $X$ 是确定性的
  2. 极值性：当 $X$ 有 $|\mathcal{X}|$ 个等可能取值时，$H(X)$ 达到最大值 $\log |\mathcal{X}|$
  3. 凹性：$H(X)$ 是概率分布 $p$ 的凹函数
二值熵函数：对于伯努利随机变量 $X \sim \mathrm{Bern}(p)$，$H(X) = -p\log_2 p - (1-p)\log_2(1-p) \triangleq H_2(p)$。当 $p = 0.5$ 时 $H_2$ 取最大值 1 bit；当 $p = 0$ 或 $p = 1$ 时 $H_2 = 0$。

联合熵与条件熵

对于两个随机变量 $X$ 和 $Y$，联合熵衡量它们联合的不确定性：

$$H(X, Y) = -\sum_{x,y} p(x,y) \log p(x,y)$$

条件熵衡量"已知 $Y$ 后，$X$ 还剩多少不确定性"：

$$H(X|Y) = -\sum_{x,y} p(x,y) \log p(x|y) = \mathbb{E}_{(X,Y)}[-\log p(X|Y)]$$

联合熵和条件熵之间有一个非常自然的链式法则：

$$H(X, Y) = H(Y) + H(X|Y) = H(X) + H(Y|X)$$

这条公式的直觉：$X$ 和 $Y$ 的总不确定性，等于先观察 $Y$ 的不确定性，再加上知道 $Y$ 之后 $X$ 还残留的不确定性。

链式法则可以推广到 $n$ 个随机变量：

$$H(X_1, X_2, \ldots, X_n) = \sum_{i=1}^{n} H(X_i | X_{1}, \ldots, X_{i-1})$$

Part 2 · 信息关联
互信息

互信息（Mutual Information）度量两个随机变量之间的信息共享量：知道 $Y$ 能减少多少关于 $X$ 的不确定性？

$$I(X; Y) = H(X) - H(X|Y) = H(Y) - H(Y|X) = H(X) + H(Y) - H(X,Y)$$

展开定义：

$$I(X; Y) = \sum_{x,y} p(x,y) \log \frac{p(x,y)}{p(x)p(y)} = D_{KL}(p(x,y) \| p(x)p(y))$$

这个展开揭示了互信息的一个深刻含义：它就是联合分布 $p(x,y)$ 与"假设 $X$、$Y$ 独立"时的乘积分布 $p(x)p(y)$ 之间的 KL 散度。换句话说，互信息衡量的是 $X$ 和 $Y$ 偏离独立性的程度。

互信息的性质

性质表述直觉
非负性$I(X;Y) \geq 0$知道 $Y$ 不可能增加对 $X$ 的不确定性
对称性$I(X;Y) = I(Y;X)$$X$ 关于 $Y$ 的信息等于 $Y$ 关于 $X$ 的信息
上界$I(X;Y) \leq \min(H(X), H(Y))$共享信息不可能超过任一变量的总信息
独立性$I(X;Y) = 0 \iff X \perp Y$独立的变量之间没有共享信息

条件互信息与数据处理不等式

给定 $Z$ 的条件下，$X$ 和 $Y$ 之间的互信息：

$$I(X;Y|Z) = H(X|Z) - H(X|Y,Z)$$

互信息的链式法则：

$$I(X_1, X_2; Y) = I(X_1; Y) + I(X_2; Y|X_1)$$

数据处理不等式（Data Processing Inequality）是信息论中最深刻的结论之一。如果 $X \to Y \to Z$ 构成马尔可夫链，则：

$$I(X; Z) \leq \min(I(X; Y), I(Y; Z))$$

直觉：对数据做任何后处理（不管是确定性变换还是随机扰动）都不可能创造新的信息。信号经过越多的处理步骤，保留的原始信息只会越少。这一定理直接否定了"通过后期处理增强信号"的幻想。

Part 3 · 无损压缩
信源编码定理

信源编码定理（Shannon 第一定理）建立了信息熵与无损压缩之间的精确关系。它回答了一个根本问题：一个信源产生的消息，最少需要多少比特才能无损表示？

Kraft 不等式

在讨论编码之前，我们需要知道什么样的码是"合法"的。对于二进制前缀码（没有任何码字是另一个码字的前缀），Kraft 不等式给出了充要条件：

$$\sum_{x \in \mathcal{X}} 2^{-L(x)} \leq 1$$

其中 $L(x)$ 是符号 $x$ 的码字长度。等号成立时，编码是完备的（没有剩余空间给新符号）。

数值例子：假设有 4 个符号，码长分别为 $L = \{1, 2, 3, 3\}$，则 $\sum 2^{-L} = 1/2 + 1/4 + 1/8 + 1/8 = 1$。这是一个完备的前缀码，对应码字 $\{0, 10, 110, 111\}$。

Shannon 第一定理

定理（信源编码定理）：设 $X$ 是离散无记忆信源，其熵为 $H(X)$。对 $X$ 的任意唯一可解码 $C$，平均码长满足：

$$H(X) \leq \mathbb{E}[L(X)] < H(X) + 1$$

更一般地，对 $n$ 个 i.i.d. 符号的分组编码，平均每符号码长满足：

$$H(X) \leq \frac{\mathbb{E}[L(X_1, \ldots, X_n)]}{n} < H(X) + \frac{1}{n}$$

当 $n \to \infty$ 时，每符号的平均码长可以无限逼近 $H(X)$，但永远不可能低于 $H(X)$。这就是信息熵的物理含义：熵是信源每符号所需的最少比特数。

证明思路

下界证明利用了 KL 散度的非负性。设 $q(x) = 2^{-L(x)} / C$（$C = \sum 2^{-L(x)}$），由 Gibbs 不等式：

$$H(X) \leq -\sum p(x) \log q(x) = \mathbb{E}[L(X)] \log 2 - \log C \leq \mathbb{E}[L(X)]$$

上界证明直接构造：令 $L(x) = \lceil -\log_2 p(x) \rceil$，则 $2^{-L(x)} \leq p(x)$，Kraft 不等式自然满足，而 $\mathbb{E}[L] < H(X) + 1$。

Part 4 · 熵编码实现
霍夫曼编码与算术编码

信源编码定理告诉我们理论下界是 $H(X)$，那么实际中如何构造逼近这个下界的编码方案？最经典的两类熵编码算法是霍夫曼编码和算术编码。

霍夫曼编码

霍夫曼编码（1952）是一种最优前缀码构造算法。它的核心思想：概率大的符号分配短码字，概率小的符号分配长码字。

算法步骤：

  1. 统计每个符号的频率，构建频率表
  2. 将所有符号放入优先队列（最小堆）
  3. 每次弹出两个频率最小的节点，合并为一个新节点
  4. 新节点的频率 = 两个子节点频率之和
  5. 重复直到只剩一个节点（根节点）
  6. 从根到叶的路径就是码字（左分支 = 0，右分支 = 1）

对应的数学保证：对符号 $x$，霍夫曼编码给出的码长 $L(x)$ 满足：

$$-\log_2 p(x) \leq L(x) \leq -\log_2 p(x) + 1$$

以下是霍夫曼编码的核心实现——树的构建部分：

import heapq
import collections
import math

class HuffmanCoding:
    """霍夫曼编码：构建最优前缀码"""

    class Node:
        def __init__(self, symbol=None, freq=0, left=None, right=None):
            self.symbol = symbol  # 叶节点有 symbol，内部节点为 None
            self.freq = freq      # 该节点的累积频率
            self.left = left      # 左子树（编码 0）
            self.right = right    # 右子树（编码 1）

        def __lt__(self, other):
            return self.freq < other.freq  # 优先队列按频率排序

    def build_huffman_tree(self, freq_table):
        """
        构建霍夫曼树。
        数学对照：每次合并两个最小频率节点 f_i, f_j，
        新节点频率 = f_i + f_j，这对应于
        L(x) 取决于 x 被合并的轮次（深度）。
        """
        heap = [self.Node(symbol=s, freq=f) for s, f in freq_table.items()]
        heapq.heapify(heap)

        while len(heap) > 1:
            left = heapq.heappop(heap)   # 频率最小的节点
            right = heapq.heappop(heap)  # 频率次小的节点
            merged = self.Node(freq=left.freq + right.freq,
                               left=left, right=right)
            heapq.heappush(heap, merged)

        return heap[0] if heap else None

    def generate_codes(self, node, current_code=""):
        """
        递归生成码表。
        数学对照：码字长度 L(x) 满足 -log2 p(x) <= L(x) < -log2 p(x) + 1
        即 Kraft 不等式的边界条件。
        """
        codes = {}
        if node is None:
            return codes
        if node.symbol is not None:
            codes[node.symbol] = current_code
            return codes
        codes.update(self.generate_codes(node.left, current_code + "0"))
        codes.update(self.generate_codes(node.right, current_code + "1"))
        return codes

接下来是编码/解码与压缩效率分析：

    def encode(self, data, codes):
        """
        编码数据。
        数学对照：总码长 = Σ L(x_i)
        平均码长 = Σ p(x) * L(x)，满足 H(X) <= E[L] < H(X) + 1
        """
        return "".join(codes[symbol] for symbol in data)

    def decode(self, encoded_bits, tree):
        """
        解码：沿霍夫曼树从根走到叶，每读一个 bit 走一步。
        前缀码保证唯一可解码性（Kraft 不等式取等号时为完备码）。
        """
        decoded = []
        node = tree
        for bit in encoded_bits:
            node = node.left if bit == "0" else node.right
            if node.symbol is not None:
                decoded.append(node.symbol)
                node = tree
        return decoded

    def compute_stats(self, data, codes):
        """
        计算压缩统计信息。
        效率 = H(X) / E[L]，越接近 1 越好。
        """
        freq = collections.Counter(data)
        n = len(data)
        entropy = -sum((c / n) * math.log2(c / n) for c in freq.values())
        avg_len = sum(freq[s] / n * len(codes[s]) for s in freq)
        return {
            'entropy': entropy,
            'avg_code_length': avg_len,
            'efficiency': entropy / avg_len if avg_len > 0 else 0,
        }
数值验证：对数据 $[3,3,3,3,2,2,2,1,1,0]$，概率分布为 $p(3)=0.4, p(2)=0.3, p(1)=0.2, p(0)=0.1$。熵 $H \approx 1.846$ bit。霍夫曼编码给出码长 $L(3)=1, L(2)=2, L(1)=3, L(0)=3$，平均码长 $= 1.9$ bit。效率 $\approx 97.2\%$。

算术编码

霍夫曼编码为每个符号分配整数长度的码字，这导致了"每符号浪费最多 1 bit"的上界。算术编码打破了这一限制：它将整个消息编码为 $[0, 1)$ 区间内的一个浮点数，理论上可以无限逼近信源熵。

核心思想：根据符号的概率分布，递归地将 $[0, 1)$ 区间按比例划分。每处理一个符号，当前区间就缩小为该符号对应的子区间。

数学上，设累积分布 $F(x) = \sum_{x' < x} p(x')$，编码过程为：

$$\text{low}_\text{new} = \text{low} + (\text{high} - \text{low}) \cdot F(x)$$

$$\text{high}_\text{new} = \text{low} + (\text{high} - \text{low}) \cdot (F(x) + p(x))$$

class ArithmeticCoding:
    """
    算术编码：将整个消息编码为 [0,1) 内的一个数。
    数学对照：编码所需比特数 ≈ -log2 p(消息)，
    即接近信源熵的理论下界，不受"整数码长"的限制。
    """

    def build_probability_table(self, data):
        """构建累积概率表 F(x) = Σ_{x'<x} p(x')"""
        freq = collections.Counter(data)
        total = len(data)
        cum_prob = 0.0
        prob_table = {}
        for symbol in sorted(freq.keys()):
            prob = freq[symbol] / total
            prob_table[symbol] = (cum_prob, cum_prob + prob)
            cum_prob += prob
        return prob_table

    def encode(self, data, prob_table):
        """
        编码：递归缩小区间。
        所需比特数 ≈ ceil(-log2(range_final))
        """
        low, high = 0.0, 1.0
        for symbol in data:
            rng = high - low
            sym_low, sym_high = prob_table[symbol]
            low = low + rng * sym_low
            high = low + rng * (sym_high - sym_low)
        return (low + high) / 2

    def decode(self, value, prob_table, length):
        """解码：反过程——根据编码值落在哪个子区间来确定符号"""
        decoded = []
        for _ in range(length):
            for symbol, (lo, hi) in sorted(prob_table.items()):
                if lo <= value < hi:
                    decoded.append(symbol)
                    value = (value - lo) / (hi - lo)
                    break
        return decoded

自适应与上下文编码

标准霍夫曼编码需要预先知道信源的概率分布。当概率分布未知或随时间变化时，我们使用自适应霍夫曼编码（动态更新频率表和树结构）；当符号之间存在相关性（如自然语言中 "q" 后几乎总是 "u"）时，我们使用上下文编码。

class ContextArithmeticCoder:
    """
    基于上下文的算术编码。
    数学：条件熵 H(X|X_{-k:-1}) ≤ H(X)
    利用上下文建模，逼近条件熵而非边际熵，获得更高压缩率。
    """

    def __init__(self, context_size=2):
        self.context_size = context_size
        self.context_models = {}

    def encode_sequence(self, data):
        """根据上下文动态切换概率模型"""
        arith = ArithmeticCoding()
        context = tuple([0] * self.context_size)
        encoded = []

        for symbol in data:
            prob_table = self.context_models.get(context,
                          self._uniform_table(data))
            value = arith.encode([symbol], prob_table)
            encoded.append(value)
            self._update_model(context, symbol)
            context = tuple(list(context[1:]) + [symbol])
        return encoded

    def _update_model(self, context, symbol):
        if context not in self.context_models:
            self.context_models[context] = collections.Counter()
        self.context_models[context][symbol] += 1
压缩率对比：对具有局部相关性的序列，上下文编码利用"1 之后更可能再出现 1"这样的局部模式。条件熵 $H(X|\text{context})$ 通常显著小于边际熵 $H(X)$，因此上下文编码的压缩率更高。这也是现代压缩算法（如 PPM、PAQ）的核心思路。
Part 5 · 信道模型
信道容量

信道容量（Channel Capacity）定义了在给定噪声条件下，信道能可靠传输信息的最大速率。

定义

离散无记忆信道的容量定义为输入 $X$ 和输出 $Y$ 之间互信息的最大值：

$$C = \max_{p(x)} I(X; Y)$$

最大化在所有可能的输入分布 $p(x)$ 上进行。信道容量只取决于信道的转移概率 $p(y|x)$，与具体输入分布无关。

Shannon-Hartley 定理

对于带宽为 $B$ Hz、信噪比为 SNR 的加性高斯白噪声（AWGN）信道：

$$C = B \log_2(1 + \mathrm{SNR}) \quad \text{bit/s}$$

其中 $\mathrm{SNR} = S / (N_0 B)$，$S$ 是信号功率，$N_0$ 是噪声功率谱密度。这一定理表明：

  • 带宽和信噪比共同决定信道容量
  • 信噪比翻倍只增加约 1 bit/s/Hz 的容量（对数增长）
  • 即使信噪比很低，只要大于零，信道容量就大于零
数值例子：电话线带宽 $B = 3000$ Hz，SNR = 30 dB（即 $\mathrm{SNR} = 1000$），则 $C = 3000 \times \log_2(1001) \approx 29,900$ bit/s ≈ 30 kbps。这就是传统调制解调器的理论上限。

常见离散信道

信道类型转移概率容量

Part 6 · 可靠传输
信道编码定理

Shannon 第二定理是信息论中"最令人惊讶的结果之一"（Cover & Thomas 语）：只要传输速率 $R < C$，就存在编码方案使错误概率任意小。

定理（有噪信道编码定理）：对于容量为 $C$ 的离散无记忆信道：

  • 若 $R < C$，则存在码率为 $R$ 的编码方案，使最大错误概率 $P_e^{(n)} \to 0$（当码长 $n \to \infty$）
  • 若 $R > C$，则任何码率为 $R$ 的编码方案，其错误概率 $P_e^{(n)}$ 有大于零的下界

Shannon 使用随机编码证明了可达性：

  1. 从所有可能的码字中随机选择 $2^{nR}$ 个码字
  2. 解码时采用联合典型性解码
  3. 当 $n$ 足够大时，随机码本的平均错误概率趋于 0
  4. 既然平均错误概率趋于 0，至少存在一个具体的码本其错误概率趋于 0

错误概率的指数衰减行为由可靠性函数 $E(R)$ 描述：

$$P_e \leq 2^{-nE(R)}, \quad E(R) > 0 \text{ 当 } R < C$$

Shannon 定理的工程意义

Shannon 的证明是存在性证明，不是构造性证明。他证明了"好的编码方案存在"，但没说怎么找。这推动了半个多世纪的编码理论研究：从 Hamming 码、Reed-Solomon 码、卷积码，到 Turbo 码（1993）和 LDPC 码（被重新发现），最终在 5G 时代逼近了 Shannon 极限。

Part 7 · 有损压缩
率失真理论

信源编码定理讨论的是无损压缩。但在很多实际场景（图像、音频、视频压缩）中，我们允许一定程度的失真，以换取更低的比特率。率失真理论（Rate-Distortion Theory）建立了这种权衡的数学框架。

定义

给定失真度量 $d(x, \hat{x})$（如 MSE）和允许的最大平均失真 $D$，率失真函数定义为：

$$R(D) = \min_{p(\hat{x}|x): \mathbb{E}[d(X,\hat{X})] \leq D} I(X; \hat{X})$$

$R(D)$ 给出了在失真不超过 $D$ 的条件下，每个符号所需的最少比特数。

关键性质

  • $R(0) = H(X)$：零失真要求下，退化为无损压缩（等于信息熵）
  • $R(D)$ 是 $D$ 的非增凸函数
  • 当 $D \geq D_{\max}$ 时，$R(D) = 0$：失真已经足够大，不需要任何比特

高斯信源的率失真函数

对于方差为 $\sigma^2$ 的高斯信源，在 MSE 失真度量下：

$$R(D) = \frac{1}{2}\log_2 \frac{\sigma^2}{D}, \quad 0 \leq D \leq \sigma^2$$

比特率与 $\log(\text{方差}/\text{失真})$ 成正比。如果要求重建质量高一倍（$D$ 减半），只需要多 0.5 bit。

与图像压缩的联系：率失真理论是 JPEG、H.264/265 等有损压缩标准的理论基础。在深度学习中，VAE 和神经编解码器的训练目标也是在做率失真优化。
Part 8 · 分布距离
KL 散度与交叉熵

KL 散度

KL 散度（Kullback-Leibler Divergence）衡量用分布 $Q$ 近似真实分布 $P$ 时，损失了多少信息：

$$D_{KL}(P \| Q) = \sum_{x} P(x) \log \frac{P(x)}{Q(x)} = \mathbb{E}_{X \sim P}\left[\log \frac{P(X)}{Q(X)}\right]$$

从信息论角度，KL 散度可以理解为：如果真实分布是 $P$，但编码方案是为 $Q$ 优化的，那么每个符号平均多用了多少比特。

性质：

  1. 非负性（Gibbs 不等式）：$D_{KL}(P \| Q) \geq 0$，等号成立当且仅当 $P = Q$
  2. 非对称性：$D_{KL}(P \| Q) \neq D_{KL}(Q \| P)$
  3. 凸性：$D_{KL}(P \| Q)$ 关于 $(P, Q)$ 联合凸

交叉熵

交叉熵衡量使用分布 $Q$ 编码来自分布 $P$ 的数据时，平均需要的比特数：

$$H(P, Q) = -\sum_{x} P(x) \log Q(x)$$

交叉熵、熵和 KL 散度之间有一条关键的关系链：

$$H(P, Q) = H(P) + D_{KL}(P \| Q)$$

直觉解读：用 $Q$ 编码 $P$ 的数据，需要的总比特数 = $P$ 自身的最少比特数（$H(P)$）+ 因为用错了分布而多花的比特数（$D_{KL}$）。

为什么机器学习用交叉熵作为损失函数？

在分类任务中，真实标签分布 $P$ 是 one-hot 向量。最小化交叉熵损失：

$$\mathcal{L} = -\sum_{i} y_i \log \hat{y}_i = H(P, Q)$$

由于 $H(P)$ 对给定数据集是常数，最小化 $H(P, Q)$ 等价于最小化 $D_{KL}(P \| Q)$，即让模型分布 $Q$ 尽可能接近真实分布 $P$。这就是交叉熵损失的深层信息论含义。

Part 9 · 连续推广
微分熵

信息熵 $H(X)$ 只适用于离散随机变量。对于连续随机变量，我们使用微分熵（Differential Entropy）：

$$h(X) = -\int_{-\infty}^{\infty} f(x) \log f(x) \, dx$$

微分熵可以为负

与离散熵不同，微分熵可以为负值。微分熵本身不直接表示"比特数"，它只在相对比较（如互信息、KL 散度）中有意义。

高斯分布的微分熵

对于 $X \sim \mathcal{N}(\mu, \sigma^2)$：

$$h(X) = \frac{1}{2} \log_2(2\pi e \sigma^2)$$

高斯分布在所有方差为 $\sigma^2$ 的连续分布中具有最大的微分熵。这意味着高斯噪声是"最难压缩"的噪声。

连续随机变量的互信息

互信息在连续情形下仍然非负：

$$I(X;Y) = h(X) - h(X|Y) = h(Y) - h(Y|X) = h(X) + h(Y) - h(X,Y) \geq 0$$

尽管 $h(X)$ 和 $h(Y)$ 可能分别为负，但 $I(X;Y)$ 始终非负。

Part 10 · 前沿应用
信息论在机器学习中的应用

变分推断与 ELBO

变分自编码器（VAE）的训练目标 ELBO（Evidence Lower Bound）可以直接用信息论语言表述：

$$\log p(x) \geq \mathbb{E}_{q(z|x)}[\log p(x|z)] - D_{KL}(q(z|x) \| p(z))$$

其中 $q(z|x)$ 是编码器（推断后验），$p(z)$ 是先验。第一项是重建质量（类似于负失真），第二项是 KL 散度正则化（类似于率）。VAE 本质上在做率失真优化。

信息瓶颈理论

信息瓶颈（Information Bottleneck）方法寻找一个表示 $T$，在压缩输入 $X$ 的同时保留与目标 $Y$ 相关的信息：

$$\min_{p(t|x)} I(X; T) - \beta \cdot I(T; Y)$$

其中 $\beta > 0$ 控制压缩与预测之间的权衡。这个框架被用来理解深度神经网络的学习过程。

互信息估计（MINE）

互信息神经估计（Mutual Information Neural Estimation, MINE）利用 KL 散度的 Donsker-Varadhan 表示来用神经网络估计互信息：

$$I(X;Y) \geq \sup_{T: \Omega \to \mathbb{R}} \mathbb{E}_P[T] - \log \mathbb{E}_Q[e^T]$$

其中 $T$ 是一个参数化的神经网络，$P$ 是联合分布，$Q$ 是边际乘积分布。
