RL 的四大新范式
2026-05 · arXiv

本次调研覆盖 4 篇重量级 Survey，从不同角度切入 RL 的前沿演进：

GFCR — LLM RL 的 Rollout 生命周期

Rollout 是 LLM RL 的数据生成核心，但长期被当作"实现细节"underreport。GFCR 将其拆解为 Generate → Filter → Control → Replay 四个模块，提供统一词汇。

世界模型 — 机器人学习的预测表示

世界模型是环境动态的预测表示，成为机器人学习核心组件。从 decoupled 到 unified VLA，从想象生成到 foundation-scale 视频世界模型。

Agentic Reasoning — 从 LLM 到自主 Agent

Agentic Reasoning 范式：LLM 作为自主 Agent，通过持续交互进行规划、行动、学习。按 Foundational → Self-evolving → Collective 三层组织。

COAML — 组合优化 + 机器学习

Predict-then-Optimize 范式的根本缺陷：预测精度 ≠ 决策质量。COAML 将组合优化预言机嵌入学习管道，实现端到端决策优化。

Survey #1 · GFCR: LLM RL Rollout 策略全景
Survey #1
Generate, Filter, Control, Replay
arXiv:2605.02913 · 47 pages · 2026-04-07

GFCR（Generate-Filter-Control-Replay）是首个系统组织 LLM RL 后训练中 rollout 策略的综述框架。作者来自 UCSD + Adobe Research + UIUC + UToronto。

核心洞察

Rollout 设计往往被当作实现细节，导致复现困难。作者指出：优化器只能从给它数据中学习，而现代后训练中，数据由模型自己通过 rollout 在线产生——所以 rollout 设计往往既支配训练成本，又决定学习信号质量。

GFCR 四阶段框架

模块功能核心子组件
G · Generate候选轨迹生成拓扑与交互结构、引导与脚手架、采样与探索
F · Filter中间信号构建结构有效性检验、正确性验证（verifier）、过程质量（PRM）、比较评估（judge/ranking）、学习价值信号
C · Control计算分配与决策Prompt/任务选择、预算调度、长度/温度配置、早停与续接、分支与剪枝、On/Off-Policy 控制
R · Replay跨 rollout 复用响应重采样与保留、片段重组合、自演化课程（无需权重更新的自主任务生成）

GFCR 符号体系

核心符号：
~x~ D — prompt/任务输入分布
~π_θ~ — 语言模型/策略
~τ = (x, u₁:T, o₁:T)~ — rollout 轨迹（含中间推理步骤和工具调用）
~T(x)~ = {τ⁽¹⁾, ..., τ⁽ᴷ⁾} — rollout group（从同一 prompt 采样 K 条轨迹）
~φ(τ⁽ⁱ⁾; T(x))~ — Filter 中间信号（有效性、验证结果、过程评分等）
~S(τ⁽ⁱ⁾; T(x))~ — 训练监督信号
~c(τ)~ — rollout 成本（token数、工具调用数、wall-clock）
~q_θ,GFCR(T | x, B)~ — 完整 GFCR pipeline 诱导的分布

关键方法分类

Filter 层的 Verifiers & Judges：

  • Verifiable rewards (Math/Code/SQL) — 单元测试执行结果作为过滤信号（RLTF、BIRD、SWEBench）
  • Process Reward Model (PRM) — 步骤级评分（Lightman 2023），需要 step-delimited rollouts
  • LLM-as-Judge — 定性判断（MT-Bench），受 prompt 设计和位置偏差影响（需 rubric-aware mitigation）

Control 层的自适应计算：

  • VCRL — 奖励方差作为难度代理，优先采样中间方差的 prompt（靠近决策边界）
  • MMR1 — 多模态 RL，结合奖励方差和轨迹多样性稳定 GRPO
  • AR3PO — 响应复用 + 自适应采样分配
  • VADE — Beta 分布建模 prompt 难度，Thompson 采样最大化信息增益

Replay 层的自演化：

  • Self-evolving curricula — 无需权重更新，自主生成新训练任务和数据
  • Recomposition — 复用验证过的片段重组成新轨迹
  • Stored buffers — 缓存正确响应，减少重复采样成本

诊断索引（GFCR Diagnostic Index）

作者提供了 practitioner-oriented 的诊断索引，将常见 rollout 病理学映射到 GFCR 模块 + 缓解杠杆：

症状模块缓解策略
全部失败（all-failed groups）G + FICL seeding (XRPO)、Step-wise judge (Stepwise Guided PO)
高方差梯度CSemEnt (SEED-GRPO)、adaptive smoothing (PARE)
teacher 能力触顶F + RVPD 变分 EM 自演化
early truncation 截断有效推理CPartial rollouts、early-exit criteria

应用域分类

领域反馈类型代表方法
Math (数学推理)可验证答案GRPO+verifier、PRM、Tree search
Code/SQL (代码/SQL)测试执行反馈RLTF、BIRD、SWEBench
Multimodal (多模态)多模态 judgeMMR1、VADE
Tool-using agents工具输出VerlTool、Doc-React

Survey #2 · 世界模型与机器人学习
Survey #2
World Model for Robot Learning
arXiv:2605.00080 · 43 pages · 2026-04-30

世界模型是环境动态的预测表示，已成为机器人学习的核心组件。作者阵容强大：Pieter Abbeel、Jitendra Malik、Yilun Du 等挂名，GitHub 持续更新。

核心问题

现有文献在架构、功能角色、体现代应用领域上高度碎片化。本综述从三个视角统一：（1）世界模型如何与策略耦合；（2）如何作为学习型模拟器；（3）机器人视频世界模型如何从想象发展到可控/结构化/基础规模。

策略耦合分类法

风格说明代表工作
Decoupled视频生成模型 + 独立策略UniPi, VidMan, Vidar, Gen2Act
Single-backbone统一视觉-动作模型VLA 系列
Latent Dynamics潜空间动态模型Dreamer系列, SAC, TDM
Unified VLA端到端视觉-语言-动作基础模型驱动

世界模型作为模拟器

  • 策略后训练（Policy fine-tuning）— 世界模型提供低成本模拟环境
  • 评估（Evaluation）— 可在模拟器中验证策略，无需真实机器人
  • 数据生成（Data generation）— 合成多样化训练数据
  • 共演化（Co-evolution）— 世界模型与策略联合优化

视频世界模型能力层级

能力演进路径：
Imagination-based generation（想象式生成）
→ Controllable（可控生成）
→ Structured（结构化）
→ Foundation-scale（基础模型规模）

三大研究方向

1. 基于模型的 RL（Model-Based RL）：世界模型提供低成本 rollout，降低真实环境交互成本。

2. 学习型模拟器（Learned Simulators）：视频生成模型驱动，绕过物理模拟器的简化假设限制，实现逼真、灵活的仿真。

3. VLA（Vision-Language-Action）：端到端视觉-语言-动作统一策略，连接视觉感知、语义理解和动作执行。

Survey #3 · Agentic Reasoning
Survey #3
Agentic Reasoning for Large Language Models
arXiv:2601.12538 · 135 pages · 2026-01-18

135 页超长综述，47 位作者联合撰写，系统将 LLM 重构为通过持续交互进行规划、行动、学习的自主 Agent。

定义：什么是 Agentic Reasoning？

传统 LLM 在封闭世界推理中表现出色，但在开放结尾和动态环境中举步维艰。Agentic Reasoning 范式将 LLM 重构为自主 Agent，通过持续交互进行规划、行动和学习——从 closed-world 走向 open-ended。

三层维度组织

层次内容核心能力
Foundational基础 Agentic 推理单 Agent 规划、工具使用、搜索（稳定环境）
Self-evolving自演化 Agentic 推理通过反馈、记忆、适应精炼能力
Collective集体多 Agent 推理协调、知识共享、共同目标

In-context vs Post-training

In-context reasoning：通过结构化编排扩展测试时交互（CoT、ReAct、Plan-then-act 等）。

Post-training reasoning：通过 RL 和 SFT 优化行为（GRPO、PPO、DPO 等）。

关键区分：In-context 扩展测试时交互规模；Post-training 优化模型自身行为。两者可叠加使用（SDAR 正是此类）。

应用场景

  • Science — 科学发现与实验设计
  • Robotics — 机器人规划与控制
  • Healthcare — 医疗诊断与决策支持
  • Autonomous Research — 自动化科研流程
  • Mathematics — 数学推理与证明

评估与诊断

  • Skill induction — 技能归纳能力
  • Library management — 工具/库管理
  • Cross-task transfer — 跨任务迁移
  • Long-horizon interaction — 长时交互
  • Governance — 真实世界部署治理

开源资源：https://github.com/weitianxin/Awesome-Agentic-Reasoning（97 节，27 图，8 表）

Survey #4 · COAML 组合优化与 ML
Survey #4
Combinatorial Optimization Augmented Machine Learning
arXiv:2601.10583 · 35 pages · 2026-01-15

COAML 来自 TU Munich / Inria / ENPC，将组合优化预言机嵌入学习管道，构建既数据驱动又保持可行性的决策策略。

核心问题：Predict-then-Optimize 的根本缺陷

PTO（先预测后优化）优化预测精度，但预测精度 ≠ 决策质量。决策质量取决于下游优化问题的结构，而非中间预测误差。

解决思路：端到端决策优化

将 CO 求解器作为神经网络的可微分层，实现端到端训练：

  • 学习目标直接对齐下游决策质量
  • 不再优化中间代理指标（如预测误差）
  • 保持解的可行性约束

范式对比（Table 1 核心表格）

范式在线计算一致性学习方式损失目标
Deterministic CO求解标称实例YesNoneN/A
Stochastic CO求解 SAAYes生成式N/A
PTO求解预测实例NoDAPrediction Error
Smart-PTO求解预测实例YesDFRegret / Cost
COAML求解CO预言机YesEnd-to-endDecision Loss

学习范式三分类

范式说明关键方法
ECM（经验代价最小化）直接优化决策质量End-to-end decision loss
SL（模仿学习）从专家轨迹学习Imitation learning
RL（强化学习）通过交互学习最优决策Policy optimization

分类法维度

  • 不确定性形式 — 静态 / 动态 / 随机
  • 决策结构 — 离散 / 连续 / 混合

应用领域

  • Scheduling — 调度优化
  • Vehicle routing — 车辆路径规划
  • Stochastic programming — 随机规划
  • Reinforcement learning — 强化学习

COAML 与 RL 的关系

COAML 中的 RL 分支将决策问题建模为马尔可夫决策过程，通过环境交互学习最优决策策略。这与 GFCR 的 rollout 策略有潜在交叉：COAML 的 "oracle" 可以类比为 GFCR 的 "verifier"，而 COAML 的 "decision loss" 对应 GFCR 的 "training signal S"。

四大 Survey 的内在联系

四篇 Survey 从不同角度切入 RL 的前沿演进，但存在深刻的内在联系：

核心连接：

GFCR 提供了 LLM RL 后训练的 rollout 生命周期框架 → 所有后训练都依赖 rollout

World Model 提供了 rollout 的"环境模拟器" → 可以降低真实环境交互成本

Agentic Reasoning 定义了"自主 Agent"的目标形态 → GFCR 的 rollout 设计服务于 Agentic 能力

COAML 提供了"决策质量直接优化"的范式 → 可以作为 GFCR Filter 的替代优化目标

四者共同指向一个趋势：RL 正在从"算法优化"走向"系统设计"，从单一目标优化走向多组件协同的端到端优化管道。
