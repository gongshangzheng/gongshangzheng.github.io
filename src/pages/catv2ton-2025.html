任务背景

虚拟换装（Virtual Try-On, VTON）旨在将服装图像"穿"到目标人物身上，生成穿着指定服装的人物图像或视频。根据输入输出形式可分为两类：

  • 图像换装（Image-based VTON）：输入服装图像 + 人物图像 → 换装后的人物图像
  • 视频换装（Video-based VTON）：输入服装图像 + 人物视频 → 换装后的人物视频，要求时序一致性

传统方法往往需要两套分离的模型——图像换装一套，视频换装一套。CatV2TON 的核心贡献是用单个扩散 Transformer 同时搞定两种任务：

  1. 单模型统一：不引入额外的 garment encoder、reference net 或 warping 模块，仅通过时间维度拼接 + 混合训练数据实现图像与视频换装统一
  2. 长视频一致性：Overlapping Clip Inference + Adaptive Clip Normalization (AdaCN)，在低资源消耗下保持长视频时序一致
  3. ViViD-S 数据集：通过朝向识别过滤 back-facing 帧 + 3D mask 平滑，净化视频换装训练数据
  4. 参数效率：超过 80% backbone 参数冻结，仅训练 attn1 和 PoseNet 即达 SOTA

相关工作

虚拟换装发展可分为两条主线：

  • 图像换装：早期以 GAN + warp-then-blend 为主（GP-VTON, DCI-VTON），扩散模型兴起后引入双流条件注入（OOTDiffusion 的 GarmentNet, IDM-VTON 的 ReferenceNet）。但这些方法只支持单帧。
  • 视频换装：核心挑战是时序一致性。ClothFormer、ViViD 等方法基于 SD UNet 构建，通过时间模块或 IP-Adapter 编码参考帧，但与图像换装是两套分离的模型。

CatV2TON 的核心洞察是：图像与视频换装的核心差异仅在于时间维度的帧数——在 DiT 框架下通过 latent 通道拼接即可统一。

数据流全景

所有输入在 VAE encode 之后，于 latent 空间通过通道拼接形成统一的 inpainting 条件：

模型架构：MMDiT 双流注意力

预训练骨架是 EasyAnimateV4 的 HunyuanTransformer3DModel。CatV2TON 裁剪了大量不必要的模块（CLIP attention、temporal attention、cross attention），只保留 attn1 层的 JointAttnProcessor2_0 用于训练：

CatV2TON 整体架构：服装与人物在 VAE latent 空间通过时间维度拼接（Temporal Concatenation）作为 DiT 条件，PoseNet 提供姿态 embedding 通过 MMDiT 双流注意力注入。

# init_transformer3d_model：裁剪无用模块
for block in transformer3d.blocks:
    block.attn_temporal = None   # 时间注意力（EasyAnimate 原有时序模块）
    block.attn_norm = None        # CLIP guided attention
    block.attn_clip = None        # CLIP attention
    block.norm_clip = None        # CLIP norm
    block.gate_clip = None        # CLIP gate
    block.norm_clip_out = None    # CLIP norm out
    block.norm2 = None            # FFN 前的第二个 norm
    block.attn2 = None            # 第二个注意力（Cross attention）

# 仅加载 attn1 模块权重
attn_blocks = torch.nn.ModuleList()
for name, param in transformer3d.named_modules():
    if "attn1" in name:
        attn_blocks.append(param)
load_checkpoint_in_model(attn_blocks, "attention/")

每个保留的 HunyuanDiTBlock 内部使用 JointTransformerBlock——这实际上是 SD3 的 MMDiT（Mixture-of-Diffusion-Transformers）架构。它采用双流设计：image tokens 和 pose tokens 各自有独立规范化和 FFN，但在 joint attention 中一起交互。

JointAttnProcessor2_0：Q/K/V 拼接

# image 投影
query      = to_q(hidden_states)
key        = to_k(hidden_states)
value      = to_v(hidden_states)

# pose 投影（encoder_hidden_states = pose_emb）
ctx_query  = add_q_proj(encoder_hidden_states)
ctx_key    = add_k_proj(encoder_hidden_states)
ctx_value  = add_v_proj(encoder_hidden_states)

# 在 seq 维度拼接 → 一次 SDPA
query = concat([query, ctx_query], dim=1)
key   = concat([key, ctx_key], dim=1)
value = concat([value, ctx_value], dim=1)

hidden_states = F.scaled_dot_product_attention(query, key, value)
# 再 split 回两部分
hidden_states, encoder_hidden_states = split(..., dim=1)

与传统 cross-attention 的本质区别：传统 CA 中 Q 来自 hidden_states、K/V 来自条件（条件是"外部注入"）；而 JointAttnProcessor 的 Q/K/V 均由 hidden 和 condition 两部分 concat 而成，两者在同一个 SDPA 操作中平等交互。双流 FFN 随后各自更新 image 和 pose 的 token 表示。

PoseNet：独立姿态编码器

class PoseNet(nn.Module):
    def __init__(self, ...):
        self.pos_embed = PatchEmbed(sample_size=32, patch_size=2, in_channels=4)
        self.time_extra_emb = HunyuanCombinedTimestepTextSizeStyleEmbedding(...)
        self.block = HunyuanDiTBlock(...)  # 单个 block，与 backbone 同架构

PoseNet 的输入是 DensePose 灰度图经 cv2.COLORMAP_VIRIDIS 转为 RGB 后的 VAE latent。输出 pose_emb 作为 DiT 每个 block 的 encoder_hidden_states 传入。

AutoMasker：多层优先级掩码合成

AutoMasker 是预处理管线核心，整合三路并行解析器：

解析器类别输出
DensePose25 部件人体表面 UV 映射 + 部件级分割
SCHP-ATR18 类ATR 人体解析
SCHP-LIP20 类LIP 人体解析（更精细的衣物分类）

掩码合成采用三级优先级设计：

  • STRONG PROTECT：面部、手部、脚部 → 绝对不遮盖。手/脚区域先通过 DensePose 定位，再与 SCHP 的 arm/leg 区域取交集，避免把附近的衣物标记为 protect。
  • WEAK PROTECT：头发、其他衣物、配件（帽子、墨镜、包、鞋、围巾）→ 尽量保留。
  • MASK AREA：根据 part 参数决定替换的衣物类别，并结合 DensePose 的对应身体部件区域。

具体步骤：先计算 mask_area = (全图 & ~weak_protect & ~背景) | dense_mask，然后 convex_hull 扩张 → 去掉 weak_protect → GaussianBlur → 阈值二值化 → 去掉 strong_protect → 膨胀。

smooth_video_mask 对视频掩码进行 3D 平滑：2D avg_pool(k=15) → permute 到 HWT → 1D avg_pool1d(k=7) → 重新二值化，消除逐帧处理引入的时序抖动。

Overlapping Clip Inference + AdaCN

长视频推理的核心挑战：GPU memory 无法容纳整段视频的一次性生成，但逐段生成又会出现边界闪烁和色彩跳变。CatV2TON 的解决方案：

  1. 将视频切分为 24 帧（像素）为一段的片段，相邻重叠 8 帧
  2. 当前片段的前 8 帧用上一片段的输出填充（sequential frame guidance）
  3. 在重叠区域用 AdaCN 对齐两个片段的分布
# AdaCN 核心：匹配重叠区 mean/std
def adacn_params(content_feat, style_feat, eps=1e-5):
    # content_feat: 当前片段重叠区输出
    # style_feat:   上一片段对应位置（作为"风格"目标）
    content_mean = content_feat.mean(dim=(2,3,4), keepdim=True)
    content_std  = sqrt(content_feat.var(dim=(2,3,4), keepdim=True) + eps)
    style_mean   = style_feat.mean(dim=(2,3,4), keepdim=True)
    style_std    = sqrt(style_feat.var(dim=(2,3,4), keepdim=True) + eps)
    return content_mean, content_std, style_mean, style_std

# 对重叠区 + 后续区均应用归一化：
result[:, :, :overlap_frames] = (overlap - c_mean) / c_std * s_std + s_mean
result[:, :, overlap_frames:] = (result - c_mean) / c_std * s_std + s_mean

AdaCN 实质上是 AdaIN（Adaptive Instance Normalization）在视频片段级别的扩展——将当前片段的分布对齐到上一片段，消除拼接边界的视觉不连续。论文消融实验证明，不使用 AdaCN 时 VFID 从 4.37 升到 4.51。

视频训练的关键技巧：模型以一定概率看到前序帧作为条件（sequential frame guidance），这使推理时能利用上一片段的输出作为 guidance。

局限性

论文在第 5 节诚实给出了局限性：

  • 分辨率限制：视频帧数量远多于图像，但 GPU memory 有限，只能生成 512x384。与图像结果相比，视频帧细节有差距。
  • 物理规律：人物运动导致的服装变形不符合物理规律（如手臂抬起时衣袖的物理变形），模型缺乏这些物理约束。

核心结论：CatV2TON 的设计哲学是"拼接即一切"——最简单的 latent 通道拼接 + MMDiT 双流注意力完成条件注入，Overlapping clip + AdaCN 解决长视频一致性，超过 80% backbone 参数冻结即达 SOTA。这与依赖双流网络（GarmentNet/ReferenceNet）或 warping 模块的传统路线形成鲜明对比。

ViViD-S 上 VFID 从 3.793（次优 ViViD）降至 1.778，DressCode 上 SSIM 0.9222 和 LPIPS 0.0367 均为最佳，证明了统一模型的性能不劣于专用模型。

收获

方法论层面

  • 通道拼接 > 独立网络：CatV2TON 证明了在 DiT 架构中，仅用通道拼接即可实现条件注入。如果你的任务涉及多模态条件注入，优先考虑通道拼接。
  • MMDiT 的双流 QKV 拼接：Q/K/V 在序列维度拼接后再 split，是 SD3 时代高效条件注入的标准模式。
  • AdaCN 的迁移潜力：AdaCN 本质上是 AdaIN 的片段级扩展，这个归一化对齐思路可复用于其他需要片段拼接的视频生成任务。

实践层面

  • 三级优先级掩码：STRONG PROTECT / WEAK PROTECT / MASK AREA 的设计是可迁移的模板，适用于需要精确区域控制的图像编辑任务。
  • Overlapping Clip + 统计归一化：GPU memory 受限时生成长视频的通用框架，不局限于 VTON。
  • 视频数据净化：back-facing 帧过滤 + 3D mask 平滑对视频生成任务至关重要，构建新数据集时优先处理。

复现注意事项

  • 视频推理 CFG=2.5，图像推理 CFG=1.0，两者不能混淆
  • sequential frame guidance 仅在视频训练中开启，图像训练无此机制
  • AdaCN 依赖 overlapping clip inference，离开 overlapping 则无效果
  • 图像训练 batch size ~64，视频训练 batch size ~4，显存规划需分别计算
