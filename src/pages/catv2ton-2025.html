任务背景

虚拟换装（VTON）旨在将服装图像"穿"到目标人物身上。传统困局：图像换装和视频换装需要两套分离的模型。CatV2TON 用单个扩散 Transformer 同时覆盖两者。

核心贡献：

  1. 单模型统一：仅 latent 通道拼接 + 混合训练，无需 GarmentNet/ReferenceNet
  2. 长视频一致性：Overlapping Clip Inference + Adaptive Clip Normalization (AdaCN)
  3. ViViD-S 数据集：朝向过滤 + 3D mask 平滑净化视频数据
  4. 参数效率：~20% 参数可训练（89.90M），超过 80% backbone 冻结即达 SOTA

模型架构

CatV2TON 整体架构：服装与人物在 latent 空间通道拼接，PoseNet 通过 element-wise addition 注入 DiT。

预训练骨架：EasyAnimateV4（HunyuanDiT, 28 blocks）。CatV2TON 做了大量裁剪：

for block in transformer3d.blocks:
    block.attn_temporal = None   # 时间注意力
    block.attn_clip = None        # CLIP cross-attention
    block.norm_clip = None        # CLIP 相关
    block.attn2 = None            # 文本 cross-attention
    block.norm2 = None            # 第二个 norm

仅保留 self-attention（attn1），可训练参数只占 ~20%（89.90M）。

PoseNet：element-wise addition（不是 MMDiT！）

这是最容易被误解的架构细节。论文 §3.1.2 原文：

"inspired by ControlNeXt, we encode DensePose sequences by duplicating the first DiT block and unlocking training, then injecting it into the network using element-wise addition after the first backbone block."

流程：DensePose 灰度图 → colormap → VAE encode → PoseNet（1 个 DiT block）→ hidden_states → 与 backbone 第一个 block 的隐藏状态 element-wise add → 继续后续 block。

这与 SD3 的 MMDiT（QKV 在 seq 维度 concat 的双流注意力）完全不是一回事。PoseNet 是一个独立模块，其输出以加法注入主干，不涉及 joint attention。

条件注入：channel 拼接

denoising 方法中，多个 latent 沿 channel 维度拼接后送入 DiT 的 inpaint_latents 输入通道：

masked_latents_cat = concat([conditioned_latents, masked_latents], dim=2)
mask_latents_cat   = concat([conditioning_mask_latents, mask_latents], dim=2)
inpaint_latents    = concat([mask_latents_cat, masked_latents_cat], dim=1)
# Shape: (b, 16, f, h, w) — 8 通道服装/人物 + 8 通道 mask

Overlapping Clip Inference + AdaCN

长视频通过重叠片段分段生成。关键参数：每个片段 24 帧（latent 6 帧），相邻重叠 8 帧（latent 2 帧）。

AdaCN 公式（AdaIN 的片段级扩展）：

xcorrected = (x − μx) / σx · σy + μy

其中 μy, σy 来自上一片段 prompt frame 特征（作为"风格"目标），μx, σx 来自当前片段。统计量在 (b, c, t, h, w) 的 (t, h, w) 维度上计算。

训练策略

三阶段 progressive training（§4.2）：

StageResolutionFramesBatchStepsDatasets
1256×1927216128KDressCode, VITON-HD, ViViD, VVT
2512×38448864KDressCode, VITON-HD, ViViD
3832×624321—DressCode, VITON-HD, ViViD-S

优化器：AdamW, lr=1e-5, constant LR, gradient clipping=1.0。硬件：4×A100 80GB。

关键技巧：

  • 10% garment dropout：训练时以 10% 概率将 garment 条件置零——实现 CFG 的无条件分支
  • 20% frame guidance：20% 样本不遮挡首帧 mask——让模型学会基于前导帧继续生成

局限性

论文 §5 诚实给出：

  • 分辨率与清晰度：视频动态变化导致清晰度不如图像结果，当前最高分辨率 832×624 仍不够
  • 物理规律：人物运动导致的服装变形缺乏物理约束，需要更大规模模型和更高质量数据
