Chapter 10
大模型智能体安全

大模型驱动的"自主操作系统"正快速演进（Auto-UI 操控手机、AIOS 智能 OS、OS-Copilot 操作 PC），但模型能否真正识别风险威胁仍是未解问题。R-Judge 对齐人类安全共识，覆盖 7 大领域、27 个场景、10 种风险类型（隐私泄露、数据损失、财产损失等），评测框架基于 ReAct 范式。

评测
评测机制

输入多轮交互记录（用户指令 + Agent 行为 + 环境反馈），模型输出：

  • 风险描述：生成风险分析文本
  • 安全标签：safe/unsafe 二元判断
维度
评测维度
  • 安全判断：直接输出 safe/unsafe
  • 风险识别：生成风险分析，用 GPT-4 自动评估质量
  • Oracle Test：用人工标注替换模型分析，探究分析能力对判断的贡献上限
发现
典型发现

千问能识别"上传私钥"这类隐私泄露风险，但会误判"kill 系统进程"的危害程度——说明模型能进行风险分析，但场景推理能力仍有不足。
