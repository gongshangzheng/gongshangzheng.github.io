二、GFCR 框架：LLM RL 的 Rollout 生命周期
GFCR · Survey #1
Generate, Filter, Control, Replay
arXiv:2605.02913 · 47 pages · 2026-04

核心问题：RL 优化器只能从给它数据中学习。现代 LLM 后训练中，数据由模型自己通过 rollout 在线产生——所以 rollout 设计往往既支配训练成本，又决定学习信号质量。但 rollout 设计长期被当作"实现细节"，导致复现困难。

图 1：GFCR 生命周期全景 — Generate-Filter-Control-Replay 四大模块的交互关系（GFCR Survey, Fig.1）

GFCR 四阶段详解

模块功能核心子组件
G · Generate候选轨迹生成拓扑与交互结构、引导与脚手架、采样与探索策略
F · Filter中间信号构建结构有效性检验、正确性验证（verifier）、过程质量（PRM）、比较评估（judge/ranking）、学习价值信号
C · Control计算分配与决策Prompt/任务选择、预算调度、长度/温度配置、早停与续接、分支与剪枝、On/Off-Policy 控制
R · Replay跨 rollout 复用响应重采样与保留、片段重组合、自演化课程（无需权重更新的自主任务生成）

GFCR 符号体系（核心定义）

GFCR 建立了一套统一符号体系来描述 rollout 管线：

  • ~x~ — prompt/任务实例，采样自数据集分布 ~D~
  • ~π_θ~ — 语言模型/策略，给定状态 ~s~ 输出 ~u_t~（tokens 或 tool/action call）
  • ~τ = (x, u_{1:T}, o_{1:T})~ — 完整 rollout 轨迹，包含 prompt、动作序列、环境反馈
  • ~φ(τ;T)~ — Filter 信号，从 rollout 中提取的中间评估信号（verifier 评分、judge 排序等）
  • ~B~ — Replay buffer，存储历史 rollout 或子轨迹供后续复用
  • ~q_{θ,GFCR}(T|x, B)~ — 完整 GFCR 管线诱导的 rollout 组分布

Generate 模块详解

Generate 指定如何为 prompt ~x~ 提议候选 rollout 集合 ~T(x) = {τ^{(i)}}_{i=1}^K~，输出作为 Filter 的输入。核心设计轴：

  • 拓扑 (Topology)：决定 rollout 的结构形态——线性链式（CoT）、树状（ReAct/MCTS）、段式（Segment Rollout）
  • 引导 (Guidance)：rollout 时的附加信息——rubrics、plans、critiques/repairs、tool traces、retrieved exemplars
  • 采样 (Sampling)：temperature、top-p、diversity beam 等探索配置
图 2：Generate 模块的三大设计轴 — 拓扑、引导、采样策略的组合空间（GFCR Survey）

Filter 模块详解

Filter 将 rollout 映射为学习信号，是 GFCR 管线中最关键的信息瓶颈。GFCR 区分了六类 Filter 机制：

Filter 类型信号类型典型方法适用场景
结构有效性 (Structural)预语义门控格式检验、执行可行性检查代码生成、SQL
正确性验证 (Correctness)硬语义门控Verifier、单元测试、符号约束Math、Code
过程质量 (Process)密集/步级信号PRM (Process Reward Model)、step-level 评分复杂推理
比较评估 (Comparative)相对排序信号Judge、LM-as-a-Judge、Bradley-Terry开放式任务
学习价值 (Learning-Value)信息性信号不确定性、困惑度、新颖性探索性任务
训练信号构建Advantage/WeightsGAE、Gradie-based selection通用 RL
图 3：Filter 模块的六类信号构建机制 — 从结构有效性到学习价值信号（GFCR Survey）

Control 模块详解

Control 在预算约束下做决策——哪些 rollout 继续、哪些终止、多少计算资源分配给哪个 prompt。核心子组件：

  • Prompt 选择：优先 rollout 哪些任务（难度估计、预期增益）
  • 预算调度：全局和 per-prompt 的计算预算分配
  • 早停 (Early Exit)：置信度高时提前终止 rollout
  • 分支与剪枝：树搜索中的枝剪策略
  • On/Off-Policy 控制：数据新鲜度与探索效率的权衡

Replay 模块详解

Replay 跨越 rollout 轮次复用数据，无需权重更新。GFCR 的 Replay 包含三类机制：

  • 响应重采样：保留高质量 rollout，在新 prompt 上复用或重放
  • 片段重组合：将多个 rollout 的子片段拼接为新轨迹
  • 自演化课程：模型自主生成新的训练任务和数据，形成自我改进循环
GFCR 与 COAML 的交叉：GFCR 的 Filter 信号（verifier/judge 评分）对齐 COAML 中的 "oracle" 概念；GFCR 的训练信号构建 (training signal construction) 对应 COAML 的 "decision loss"——两者都在解决"什么样的监督信号最有利于学习"的问题。
三、世界模型：机器人学习的预测表示
World Model Survey · Survey #2
World Model for Robot Learning
arXiv:2605.00080 · 43 pages · 2026-04

世界模型是环境动态的预测表示——给定当前状态和动作，预测下一步状态和观测。在机器人学习中，世界模型成为策略学习、规划、模拟、评估的核心组件，随基础模型和大规模视频生成的兴起而快速推进。

图 4：世界模型在机器人学习中的角色 — 从预测到规划，从模拟到评估（World Model Survey, Fig.1）

策略耦合分类法

世界模型与机器人策略的耦合方式决定了系统架构，影响训练效率和泛化能力：

耦合风格说明代表工作
Decoupled视频生成 + 独立策略（两阶段）UniPi, VidMan, Vidar, Gen2Act
Single-backbone统一视觉-动作模型VLA 系列
Latent Dynamics潜空间动态模型Dreamer, SAC, TDM
Unified VLA端到端视觉-语言-动作基础模型驱动
MoE/MoT专家世界模型 backboneMixture-of-Experts 模型
图 5：世界模型与策略的耦合分类 — 从 Decoupled 到 Unified VLA（World Model Survey）

作为模拟器的世界模型

世界模型的核心应用之一是作为可微分模拟器：

  • 策略后训练：在模拟器中进行低成本的 rollout，降低真实机器人交互成本
  • 环境评估：无需真实机器人即可评估策略质量
  • 数据生成：合成多样化的训练数据，扩充稀缺的真实机器人数据
  • 共演化：世界模型与策略联合优化，相互提升

视频世界模型的能力层级

机器人视频世界模型正从想象式生成向可控、结构化、基础模型规模演进：

  1. Imagination-based generation：给定初始帧生成未来轨迹视频
  2. Controllable generation：支持动作/条件控制的条件视频生成
  3. Structured prediction：引入几何/物理先验的结构化预测
  4. Foundation-scale：基础模型规模的大规模视频生成作为世界模型
图 6：机器人视频世界模型的能力层级演进（World Model Survey）
World Model 与 GFCR 的交叉：GFCR 的 Control 模块（计算预算调度）和 Replay 模块（数据复用）可以借助世界模型实现——在世界模型中低成本生成 rollout 候选项，从而降低真实环境交互成本。这代表了一种新型的「世界模型增强的 rollout」。
四、Agentic Reasoning：从 LLM 到自主 Agent
Agentic Reasoning Survey · Survey #3
Agentic Reasoning for Large Language Models
arXiv:2601.12538 · 135 pages · 2026-01

传统 LLM 在封闭世界推理中表现良好，但在开放结尾和动态环境中举步维艰。Agentic Reasoning 范式将 LLM 重构为通过持续交互进行规划、行动、学习的自主 Agent。

图 7：Agentic Reasoning 的核心架构 — 规划、行动、学习的三层循环（Agentic Reasoning Survey）

三层能力维度

层次能力核心方法与 GFCR 的对应
Foundational单 Agent 规划、工具使用、搜索CoT、ReAct、Plan-then-actGFCR Generate（拓扑结构）
Self-evolving反馈、记忆、适应RL post-training、Memory、AdaptationGFCR Replay（自演化课程）
Collective多 Agent 协调、知识共享Coordination、CommunicationGFCR Filter（judge 信号）

In-context vs Post-training

Agentic Reasoning 能力的获取方式分为两类：

  • In-context：通过结构化编排在测试时扩展交互（CoT、ReAct、树搜索）——无需重训练但受限于上下文窗口
  • Post-training：通过 RL/SFT 在训练时优化模型行为（GRPO、PPO、DPO）——能力更持久但需要训练
图 8：Self-evolving Agentic Reasoning — 反射反馈、参数适应、验证器反馈的三种自演化机制（Agentic Survey）

GFCR 与 Agentic Reasoning 的对应关系

Agentic Reasoning 定义了"自主 Agent"的目标形态，而 GFCR 的 rollout 设计服务于这一目标：

  • Multi-turn 交互 → GFCR Generate 的拓扑结构（树/段式 rollout 支持多步交互）
  • Tool use → GFCR Filter 的 verifier 信号（工具执行的正确性验证）
  • Self-evolution → GFCR Replay 的自演化课程（无需权重更新的自主改进）
  • Memory & Adaptation → GFCR Replay 的缓冲与重组合（历史轨迹复用）

工具使用优化的三条路线

Agentic Survey 区分了工具使用的三种优化方式：

  • In-context Tool-integration：通过 prompt 工程在测试时引导工具选择与调用
  • Post-training Tool-integration：通过 RL/SFT 让模型学会更高效的工具使用策略
  • Orchestration-based：通过编排层协调多工具的协作执行
五、COAML：决策质量端到端优化
COAML Survey · Survey #4
Combinatorial Optimization Augmented Machine Learning
arXiv:2601.10583 · 35 pages · 2026-01

核心问题：传统 Predict-then-Optimize（PTO）范式存在根本缺陷——预测精度 ≠ 决策质量。即使预测误差很小，最优决策也可能因预测误差方向的不同而完全错误。

图 9：COAML 统一框架 — 将组合优化 oracle 嵌入学习管线，实现决策质量端到端优化（COAML Survey, Fig.1）

范式对比

范式一致性学习方式损失目标核心问题
PTO (Predict-then-Optimize)❌ NoDA（直接应用）Prediction Error优化目标与决策目标不一致
Smart-PTO✅ Yes (受限)DF（端到端）Regret / Cost限于线性目标函数
COAML✅ YesEnd-to-endDecision Loss需要可微分的 CO oracle

COAML 核心思想

将组合优化求解器作为神经网络的可微分层：

  • 学习目标直接对齐下游决策质量
  • 不再优化中间代理指标（预测误差）
  • 通过端到端梯度回传训练整个管线

学习范式分类

范式说明代表方法
ECM (Empirical Cost Minimization)经验代价最小化，直接优化决策质量End-to-end 训练
SL (Supervised Learning)模仿学习，从专家轨迹学习决策策略行为克隆
RL (Reinforcement Learning)通过环境交互学习最优决策Policy Gradient
图 10：决策优化范式对比 — 从 PTO 到 Smart-PTO 到 COAML 的演进（COAML Survey）

COAML 与 GFCR 的关联

COAML 中的 "oracle" 类比 GFCR 的 "verifier"：

  • COAML 的 "decision loss" 对应 GFCR 的 "training signal"（最终学习目标）
  • COAML 的 "end-to-end training" 对应 GFCR 的 "Replay"（整个管线联合优化）
  • 两者的核心问题都是：什么样的信号最能驱动有效学习？

八、Offline RL 与 Multi-Agent RL

Offline RL：从静态数据学习

Offline RL 在仅有离线数据、无法与环境交互的情况下学习。核心挑战：

  • 分布偏移 (Distribution Shift)：策略部署时遇到训练数据未覆盖的状态-动作对
  • 外推误差 (Extrapolation Error)：对 OOD 动作的价值高估
  • 价值高估 (Overestimation)：自举法在缺少真实交互时加剧
方法核心思想
CQL (Conservative Q-Learning)保守 Q-learning，惩罚 OOD 动作的价值
IQL (Implicit Q-Learning)隐式 Q-learning，通过 expectile 回归避免显式策略提取
Model-Based Offline学习世界模型，在模型中规划以规避真实交互风险
Offline RL 与 World Model：Model-Based Offline RL 直接使用 World Model 的思路——学习环境动态模型，在模型中进行 planning。这与 World Model Survey 中的"模拟器"方向高度一致。

Multi-Agent RL：协作与竞争

多个 Agent 在共享环境中如何学习？核心挑战：

  • 非平稳性：其他 Agent 的策略在不断变化
  • 信用分配：团队奖励如何归因到个体
  • 可扩展性：Agent 数量增加时的计算复杂度
架构说明代表方法
CTDE中心化训练，分布式执行QMIX, MAPPO
Fully Decentralized完全分布式Independent PPO
Communication-based显式通信CommNet, TarMAC

  • LEMON：反事实 RL 多智能体编排
  • Adaptive TD(λ)：协作 MARL 的自适应时序差分

统一趋势

RL 正在从「算法优化」走向「系统设计」：

  • 单一目标优化 → 多组件协同的端到端管道（GFCR）
  • 孤立方法 → 模块化、可组合的框架
  • 纯环境驱动 → LLM 先验 + 世界模型 + RL 的融合
  • 离线优化 → 在线 rollout + 离线数据的混合范式
最核心的演进：从"选择哪个 RL 算法"（PPO vs DPO vs GRPO）到"设计什么样的 rollout 数据生成管线"。数据质量正在成为比算法选择更关键的差异化因素。
十、开放问题与未来方向
  1. Verifier / Judge 的校准问题：GFCR Filter 的质量直接依赖于 verifier/judge 的校准精度，但目前缺乏系统性评估方法
  2. 计算预算的最优分配：GFCR Control 模块如何在多样化任务中实现计算预算的最优分配仍是开放问题
  3. Self-evolution 的安全性：GFCR Replay 的自演化课程可能导致 reward hacking，需要 provenance tracking 机制
  4. World Model 与真实物理的 gap：视频生成模型作为世界模型时，物理真实性（接触、摩擦、刚体力学）仍是挑战
  5. COAML 的可微分 oracle 普适性：组合优化求解器的可微分化需要大量工程工作，难以自动化
  6. Agentic 的长程可靠性：Agentic Reasoning 在长时间跨度的多步交互中如何保持可靠性
