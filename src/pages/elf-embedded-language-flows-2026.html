简介

ELF（Embedded Language Flows）是 MIT 何恺明与 Jacob Andreas 团队于 2026 年 5 月提出的连续扩散语言模型。与当前主流的离散扩散模型（如 MDLM、Duo）不同，ELF 将去噪过程完全保留在连续的 embedding 空间中，只在最后一步（t=1）做 discretization。

核心结论：105M 参数的 ELF-B 在 32 步采样内达到 Gen PPL 24，优于 170M 的离散扩散模型，且训练 tokens 减少 10 倍。这不是通过更大的模型规模实现的，而是通过更简洁的架构设计——shared-weight network、无独立 decoder、无每步 token-level supervision。

核心创新点：

  • 全程驻留连续空间（t ∈ (0,1)），仅在 t=1 一步离散化
  • Shared-weight network：同一网络同时承担去噪和最终解码，无需独立 decoder
  • x-prediction 参数化：直接预测 clean embedding 而非 velocity，使 t→1 时梯度稳定
  • 训练时 CFG 实现：无需额外 forward pass，通过 batch 中的条件/无条件预测计算 guidance target
  • T5 frozen encoder embeddings：只用训练时，推理无额外模块

背景：扩散语言模型的两条路线

扩散语言模型（DLM）分为两条技术路线：

方法代表工作去噪空间Discretization 策略

连续 DLMCDCD, DiffuSeqembedding space每步有 CE supervision

连续 DLM 长期落后于离散方法，根源不在于语言"本质离散"，而在于之前方法在训练中对中间状态施加 token-level 监督（cross-entropy loss），打断了 Flow Matching 连续轨迹的灵活性。Flow Matching（Albergo & Vanden-Eijnden, 2023）的出现提供了理论工具，使连续 embedding 扩散能够真正发挥效力。

Insight：去噪轨迹的完整性决定了生成质量。之前连续 DLM 在每步都施加 token-level cross-entropy supervision，等于把连续流形强制"拽回"到离散 token 空间。ELF 只在最后一步做 discretization，去噪全程在连续空间完成，流动态获得最大灵活性。类比：图像扩散从来不在中间步骤"四舍五入到像素值"，语言扩散也不应该每步"四舍五入到 token"。
核心创新

创新 1：Shared-weight Network 无需独立 Decoder

标准的 Latent Diffusion 需要 encoder + latent space + decoder，因为图像的像素空间和 latent 空间的语义完全不同。ELF 发现：embedding 空间到 token 空间的映射，本质上是一个线性投影（unembedding matrix），与去噪过程共享参数即可。

具体做法：unembedding 权重 $W$ 与 embedding 权重绑定（tied weights），最终时间步的输出直接通过这个线性映射得到 token logits，不需要额外的神经网络 decoder。这消除了 latent diffusion 中"需要独立 decoder"这个额外模块。

创新 2：x-prediction > v-prediction

标准 Flow Matching 预测 velocity $v = x - \varepsilon$，但 ELF 选择预测 clean embedding $x$。当 $t \to 1$ 时，$z_t \to x$，所以 $v = (x - z_t)/(1-t)$ 中分子分母同时趋于 0，梯度消失。x-prediction 在整个时间域内保持稳定梯度，而且与最终 discretization 天然一致，因为两者都是在 $\mathbb{R}^d$ 空间操作。

创新 3：CFG 在连续空间几乎零成本实现

在图像扩散中，CFG 需要两个 forward pass（条件/无条件）各一次。ELF 通过 batch 内计算技巧，在训练时同时得到条件和无条件预测，直接计算 CFG target，无需额外的网络前向传播。

模型架构

输入 token 序列 → T5 frozen encoder → continuous embeddings → ELF 去噪（连续时间 Flow Matching）→ 最后一步 discretization → 输出 token 序列

从离散 Token 到 Continuous Embedding

给定句子 $s = [s_1, ..., s_L]$，通过 T5-small encoder 映射为连续 embedding $x \in \mathbb{R}^{L \times 512}$。embedding 经过 bottleneck（512 → 128 → 768）后送入 ELF Transformer。注意：embedding 只在训练时使用，推理时没有 encoder 的额外开销——这是与 Latent Diffusion 的本质区别。

Flow Matching Formulation

线性插值（rectified flow）：

$$z_t = t \cdot x + (1-t) \cdot \varepsilon, \quad \varepsilon \sim \mathcal{N}(0, I)$$

速度场定义为时间导数：

$$v = \frac{dz}{dt} = x - \varepsilon$$

训练目标是 MSE 回归：

$$\mathcal{L}_{MSE} = \mathbb{E}_{t,x,\varepsilon} \frac{1}{(1-t)^2} \| x_\theta(z_t, t) - x \|^2$$

其中 $x_\theta = net_\theta(z_t, t)$ 是网络对 clean embedding 的预测。注意 ELF 参数化 $x$ 而非 $v$，这使得 t→1 时梯度不会消失。

最后一 Step 的 Discretization

$t=1$ 时，为了提供 non-trivial 的训练信号，对 clean embedding 施加 per-token corruption：

$$\tilde{z} = p \cdot x + (1-p) \cdot \varepsilon', \quad p \sim \text{LogitNormal}(0.8, 0.8)$$

网络输出 $h = x_\theta(\tilde{z})$，再通过 unembedding 矩阵 $W$ 得到 token logits：

$$\mathcal{L}_{CE} = \mathbb{E}_{\tilde{z}}[ \text{CrossEnt}(W \cdot h, s) ]$$

消融实验与失败案例

消融实验的关键发现：

  • x-prediction 优于 v-prediction：v-prediction 在 512d embedding 上显著退化，x-prediction 保持稳定
  • CFG 有效但有饱和点：scale=3 是最佳点，超过 3 后 perplexity 反升
  • SDE > ODE（尤其少步数）：随机性减少了误差累积；在 8-16 步时差异最显著
  • Shared-weight ≈ two-stage decoder：两者整体 tradeoff 相似，但 shared-weight 在低 perplexity 区域更优
  • Pretrained contextual embedding 优于非上下文和随机 embedding：T5 frozen encoder 是最优选择

失败案例：

  • ε-prediction 完全崩溃（所有维度、所有设置）
  • v-prediction 在 embedding dim > 512 时显著退化
  • 512d bottleneck 产生极高 perplexity（必须用 128 bottleneck）
  • CFG scale > 3 后 perplexity 回升
结论与收获

ELF 证明了连续 DLM 在语言建模中可以超越离散方法，核心是"只在最后一步做 discretization"。Flow Matching 的连续时间 formulation 使得图像域的技术（CFG、SDE/ODE 采样、self-conditioning）可以零成本迁移到语言建模。

主要限制：(1) 105M 参数仍是概念验证，无 instruction tuning / RLHF 下游任务验证；(2) scaling law 尚未建立；(3) Cola DLM 同天发布说明大厂已押注这条路线。

工程收获：

  • 去噪轨迹的完整性 > 每步精确度：让连续流形保持完整比在每步精确预测 token 更重要
  • Interface 设计决定上限：Diffusion-LM 和 CDCD 的失败是因为在每步都重新"翻译"到离散空间打断了流形
  • x-prediction 在高维连续空间的重要性：标准 Flow Matching 用 v-prediction 在图像生成中 work，但在语言的高维 embedding 空间（768d）中，t→1 时的梯度消失问题变得严重
  • "无 decoder"不是没有 decoder，而是 decoder 已隐含在线性映射中
ByteDance Cola DLM vs MIT ELF 的路线对比：Cola DLM 用 Text VAE + block-causal DiT，走"分层解耦"路线；ELF 用 frozen encoder + shared-weight network，走"最简适配"路线。两者共同点是都认为连续 embedding 是正确方向，分歧在于如何处理 embedding 空间和 token 空间的接口。
