研究问题与范围

本综述围绕以下核心问题展开：

  • 视觉编码器应该如何分类？ 从监督CNN到Transformer、自监督学习、多模态编码器、Latent Tokenizers，清晰的分类揭示设计权衡。
  • 各家族有哪些归纳偏置？ 卷积局部性、全局注意力、对比对齐、离散化潜在空间——每个都编码了关于视觉结构的不同假设。
  • 自监督和多模态范式如何重塑视觉理解？ 超越标签监督开启了新的缩放定律和迁移能力。
  • 视觉编码与视觉生成的关系是什么？ Tokenizers和Latent Encoders桥接理解与生成，引发表示连续性vs离散性的问题。

范围： 时间线从2012（AlexNet）到2025，聚焦自监督和多模态学习的后2020发展。涵盖图像编码器，视频、3D、点云编码器在必要时提及但不系统覆盖。

一、监督视觉骨干网络

现代视觉编码时代始于在大规模图像分类（ImageNet ILSVRC）上训练的监督卷积网络。这些架构确立了局部连通性和平移等变性的基本归纳偏置，塑造了所有下游视觉编码范式。

AlexNet (2012)

核心论文："ImageNet Classification with Deep Convolutional Neural Networks" (Krizhevsky, Sutskever, Hinton, 2012, NeurIPS)

核心思想： AlexNet证明了一个深CNN（ReLU激活、dropout正则化、GPU加速训练）可以远超传统特征工程流程（SIFT + SVM）。架构使用5个卷积层加3个全连接层，局部响应归一化和重叠池化。

关键贡献：

  • 在ImageNet上验证了深度学习对视觉识别的可扩展性（top-5错误率16.4% vs 亚军26.2%）
  • 普及了ReLU激活和dropout作为标准训练技术
  • 确立了"ImageNet预训练+下游微调"的范式

VGG (2014)

核心论文："Very Deep Convolutional Networks for Large-Scale Image Recognition" (Simonyan, Zisserman, 2015, ICLR)

核心思想： VGG的设计原则是极度简洁：只使用3×3卷积滤波器深度堆叠，在每次池化后将通道数翻倍。这取代了AlexNet中较大的滤波器（7×7、11×11），用更小、更具判别性的操作构建等效感受野。

关键贡献：

  • 证明了深度本身是关键因素：VGG-16（16层）和VGG-19（19层）明显优于更浅的设计
  • 确立了均匀、模块化的构建块原则（3×3 conv堆叠）
  • 成为下游任务（检测、分割）的默认特征提取器

ResNet (2015)

核心论文："Deep Residual Learning for Image Recognition" (He, Zhang, Ren, Sun, 2016, CVPR)

核心思想： ResNet引入跳跃连接（残差连接），允许梯度通过恒等快捷方式直接流动，有效地使网络学习残差函数F(x) = H(x) - x而非未参考映射。这优雅地退化：如果恒等映射是最优的，网络简单地将残差推向零。

关键贡献：

  • 使前所未有的深度网络（ResNet-152及以上）的训练成为可能，解决了退化问题——更深的网络性能反而更差
  • 引入了残差学习框架作为通用架构原则，适用于视觉之外
  • 在ILSVRC 2015中以top-5错误率3.57%获胜，超越了人类在分类基准上的表现

EfficientNet (2019)

核心论文："EfficientNet: Rethinking Model Scaling for Convolutional Neural Networks" (Tan, Le, 2019, ICML)

核心思想： EfficientNet提出复合缩放，根据由神经架构搜索（NAS）确定的固定比率联合缩放网络深度、宽度和输入分辨率。而不是任意缩放一个维度，复合系数φ均匀缩放所有三个维度。

ConvNeXt (2022)

核心论文："A ConvNet for the 2020s" (Liu et al., 2022, CVPR)

核心思想： ConvNeXt是一个纯卷积网络，通过融入ViT/DeiT/Swin Transformer的设计选择（而不实际使用注意力）来现代化ResNet架构。从标准ResNet-50开始，逐步采用：更大核（7×7）、深度可分离卷积、倒置瓶颈设计（来自MobileNetV2）、LayerNorm代替BatchNorm、GELU激活、更少的激活/归一化层。

二、基于Transformer的视觉编码器

Vision Transformer（ViT）的引入挑战了卷积归纳偏置对视觉识别必要的长期假设。

ViT (2020)

核心论文："An Image is Worth 16x16 Words: Transformers for Image Recognition at Scale" (Dosovitskiy et al., 2021, ICLR)

核心思想： ViT将图像分割成固定大小的patch（通常16×16），线性嵌入每个patch，添加位置嵌入，将结果序列输入标准Transformer编码器。架构完全放弃卷积归纳偏置，将图像识别作为序列分类问题处理。

关键贡献：

  • 证明了纯Transformer可以在足够数据（JFT-300M）预训练时匹配或超越CNN在图像分类上的表现
  • 确定了预训练数据规模的关键作用：仅在ImageNet（1.3M图像）上预训练时ViT表现不如ResNet，但在更大数据集上预训练时超越它们
  • 确立了几乎所有后续视觉Transformer遵循的patch嵌入范式

DeiT (2021)

核心论文："Training data-efficient image transformers & distillation through attention" (Touvron et al., 2021, ICML)

核心思想： DeiT通过强数据增强（RandAugment、CutMix、Mixup）、正则化（Stochastic Depth、重复增强）和新型蒸馏策略，使竞争性ViT训练仅在ImageNet上成为可能（无需JFT规模数据）。

Swin Transformer (2021)

核心论文："Swin Transformer: Hierarchical Vision Transformer using Shifted Windows" (Liu et al., 2021, ICCV)

核心思想： Swin引入分层Transformer和移位窗口注意力。不是全局自注意力，它在局部窗口（7×7 patch）内计算注意力，并在连续层之间移位窗口分区以实现跨窗口信息交换。

PVT / MViT

PVT："Pyramid Vision Transformer" (Wang et al., 2021, ICCV) — 首个提出纯Transformer金字塔骨干的网络，适用于密集预测任务。

MViT："Multiscale Vision Transformers" (Fan et al., 2021, ICCV) — 通过池化注意力操作在通道维度扩展和空间分辨率收缩的同时实现多尺度架构。

三、自监督视觉表示学习

自监督学习（SSL）已成为训练视觉编码器而不依赖人类标注标签的主导范式。SSL方法通过各种前置任务从数据本身创建监督信号。

对比方法

SimCLR (2020)：通过最大化相同图像的增强视图之间的一致性来学习表示，使用NT-Xent损失。

MoCo (2020)：维护一个动量更新的负表示队列，将字典大小与小批量大小解耦。

SwAV (2020)：用基于聚类的方法替代成对对比目标，通过在线聚类消除大量负样本的需求。

非对比与自蒸馏方法

BYOL (2020)：使用两个网络（在线网络和目标网络），在线网络预测目标网络的表示，非对称架构防止表示崩溃。

DINO (2021)：将SSL框架为自蒸馏问题：学生网络学习匹配动量更新教师网络在不同增强视图上的输出分布。发现了DINO训练的ViT自发产生语义分割能力。

DINOv2 (2023)：将DINO+iBOT训练扩展到14.2亿张图像的策划数据集（LVD-142M），产生在密集预测任务上表现竞争或超越监督基线的通用视觉特征，无需微调。

掩码图像建模（MIM）

MAE (2021)：掩码极高比例（75%）的图像patch，使用轻量级非对称编码器-解码器重建原始像素值。

BEiT (2021)：使用预训练的dVAE tokenizer（来自VQGAN/DALL-E）将图像patch离散化为视觉token，然后预测掩码patch的离散token ID。

EVA (2022)：预训练ViT以重建预训练CLIP教师掩码patch token，CLIP特征作为语义丰富的重建目标。

四、多模态视觉编码器

多模态视觉编码器通过与另一种模态（通常为文本）的对齐来学习视觉表示。对比语言-图像预训练范式已成为主导方法。

CLIP (2021)

核心论文："Learning Transferable Visual Models From Natural Language Supervision" (Radford et al., 2021, ICML)

CLIP训练图像和文本的双编码器在4亿对图像-文本对上。训练目标是最大化匹配图像-文本对的余弦相似度，同时最小化非匹配对的相似度。

ALIGN / SigLIP / EVA-CLIP

ALIGN (2021)：将CLIP范式扩展到18亿对噪声图像-文本对，证明了规模和多样性可以补偿标签噪声。

SigLIP (2023)：用成对sigmoid损失替代基于softmax的对比损失，消除了对大批量同步归一化的需求。

EVA-CLIP (2023)：将MIM预训练的EVA与CLIP式对比对齐结合，产生兼具强空间理解（来自MIM）和丰富语义（来自对比对齐）的视觉编码器。

VLM中的视觉塔

视觉编码器在视觉-语言模型（VLM）中作为"眼睛"。两个关键设计选择：

  1. 使用哪种视觉编码器？大多数VLM使用CLIP ViT-L/14或SigLIP作为视觉塔。
  2. 如何连接？线性投影、Q-Former（BLIP-2）和pixel shuffle（InternVL）是常见的桥接策略。

五、视觉Tokenizers与Latent Encoders

视觉Tokenizers将图像压缩成紧凑的潜在表示，桥接视觉理解和视觉生成。这是视觉编码中发展最快的领域。

VQ-VAE (2017)

核心论文："Neural Discrete Representation Learning" (van den Oord et al., 2018, NeurIPS)

将向量量化引入变分自编码器框架。编码器产生连续潜在向量，映射到学习码本中的最近条目。离散瓶颈使自回归先验能够在量化代码上建模。

VQGAN (2021)

核心论文："Taming Transformers for High-Resolution Image Synthesis" (Esser et al., 2021, CVPR)

用对抗训练目标和感知损失（LPIPS）增强VQ-VAE。GAN判别器鼓励解码器产生感知锐利的重建，而感知损失保持结构相似性。

TiTok (2024) — 1D Tokenization的奠基工作

核心洞察：传统2D tokenizer的token数量与图像分辨率绑定（256×256 → 16×16 grid = 256 tokens），且每个token只包含局部patch信息。

TiTok的解法：编码端ViT处理patch序列 + K个可学习latent tokens，最终只保留latent tokens；量化使用标准VQ（码本4096）；解码端K个量化latent tokens + mask tokens通过MaskGIT渐进式解码；结果：256×256图像压缩为仅32个token。

关键洞察：TiTok的码本像"字母表"——单个token没有明确语义，必须拼成整体才能表意；而VQGAN的码本像"词典"——单个token有语义但组合受约束。

MAGVIT-v2 (2024)

Lookup-free Quantization (LFQ) + 非常大码本（2^18=262,144条目），在各种生成任务中实现最先进性能。核心论点是"tokenizer > generator"——更好的tokenizer设计比更复杂的生成模型更重要。

HART — 连续-离散混合

离散token（大局）+ 连续残差token（细节），仅37M参数的残差扩散模块，重建FID从2.11降至0.30。本质：离散负责序列建模（AR/LLM友好），连续负责高保真重建。

GigaTok (ICCV 2025)

30亿参数的语义正则化对齐预训练视觉编码器，decoder优先扩展。系统回答了"tokenizer变大后重建与生成为何此消彼长"——根源在于潜在空间复杂度失控。

MAETok

核心发现是"变分约束非必需，有判别性的潜在空间结构才是关键"，ImageNet生成仅用128 tokens达到gFID 1.69，训练快76倍。

SoftVQ-VAE

用软分类概率替代硬量化，桥接离散和连续的表示学习。

GNN / 超图用于视觉表示

图神经网络和超图在视觉表征学习中提供了一个有前景的方向，与传统卷积和Transformer方法正交。从ViG（NeurIPS 2022）到DVHGNN（2025）的完整演进路线，与CNN/ViT形成对比。

扩散Latent Encoders

潜在扩散模型（Stable Diffusion、DALL-E 2）使用自编码器将图像压缩到较低维潜在空间，在其中扩散更高效。质量直接决定了生成天花板和扩散过程的效率。

六、评估协议

线性探测：冻结编码器，在表示顶部训练单个线性分类器。衡量学习特征的质量。

微调：在下游任务上端到端微调编码器。衡量预训练表示的可迁移性。

k-NN评估：使用冻结编码器提取特征，在特征空间中进行k最近邻分类。直接衡量表示的语义聚类质量。

零样本迁移：对于多模态编码器（CLIP、ALIGN）：使用文本提示评估零样本分类或检索。衡量视觉和语言表示之间的对齐质量。

重建质量：对于tokenizers和latent encoders：测量rFID、PSNR、SSIM和LPIPS。衡量压缩表示的信息保留。

开放问题

  1. 理解-生成二元性：单一视觉编码器能否同时服务理解（识别、推理）和生成（合成、编辑）？当前tokenizer为一个或另一个优化。
  2. 视觉SSL的缩放定律：语言模型的缩放定律已被很好表征，视觉SSL的缩放行为（关于模型大小、数据大小和计算）理解较少。
  3. 离散vs连续表示：语言模型在离散token上操作，而最佳生成模型（扩散）在连续潜在空间中操作。能否找到在两个领域都不牺牲质量的桥接表示？
  4. 时空一致性：视觉编码器应如何在视频中处理时间一致性？当前方法要么逐帧应用图像编码器，要么引入时间注意力，但缺乏对时空编码的原则性处理。
  5. 边缘效率：向更大视觉编码器（ViT-g at 1.1B参数）的趋势与部署约束冲突。能否在不完全失去使它们有价值特性的情况下蒸馏或压缩这些编码器？
  6. Tokenizer质量作为瓶颈：最近证据表明，tokenizer质量而非生成模型架构是视觉生成的主要瓶颈。这将研究优先级从"更好的生成模型"重新定位到"更好的视觉tokenizer"。
  7. 编码-解码耦合：现有tokenizer几乎都假设编码是一次性完成的（encode once, decode later）。但ProgressiveDiTok和DiffC的思路是：编码端逐步传递信息，解码端逐步恢复，编解码过程本身就有结构。

关键结论

  • 视觉编码器领域从监督CNN经过Transformer、自监督方法、多模态对齐发展到专用tokenizers，每个范式都扩展了视觉表示的能力。
  • 自监督学习（特别是DINO/iBOT/DINOv2）在视觉编码器方面已基本弥合与监督预训练的差距，同时产生监督方法不具备的涌现特性（语义分割、以对象为中心的注意力）。
  • 视觉编码器的选择越来越依赖于任务：CLIP变体用于开放词汇理解，DINO用于密集预测，MAE预训练的ViT用于微调，以及用于生成的专用tokenizers。
  • 视觉编码的前沿正收敛于服务于理解和生成的双用途tokenizers问题，以重建保真度、语义质量和离散化为核心开放挑战。
