一、从指令开销到可重构计算

当代计算面临三重矛盾：大数据驱动的算力需求持续膨胀，摩尔定律与登纳德缩放定律双双放缓，而领域专用架构（DSA）又面临算法快速迭代的挑战。ASIC 虽然能效极高，但高昂的非重复性工程成本（NRE）和漫长的流片周期让它难以跟上深度学习模型「月月换新」的节奏。

FPGA 与 CGRA 的出现，正是在「灵活性」与「能效」之间寻找第三条道路。与 CPU、GPU 通过指令驱动执行不同，可重构硬件的本质是静态配置：上电后把一帧配置比特流（bitstream）写入芯片，后续运行时不再取指、译码、访存寄存器堆，从而彻底消除指令开销。

数据显示，CPU 数据通路中约 40% 的能耗消耗在取指、译码和寄存器文件访问上；GPU 的动态功耗中也有约 30% 被指令相关开销占据。FPGA 与 CGRA 将这些开销降为零，把全部硅面积和能量留给真正的数据计算。

二、FPGA 基础架构：LUT、CLB 与互连资源

FPGA（Field Programmable Gate Array，现场可编程门阵列）的核心思想可以用一句话概括：用预制的「海量的可重构逻辑单元 + 可编程互连网络」模拟任意数字电路。

2.1 LUT：可重构的「原子」

查找表（Look-Up Table，LUT）是 FPGA 可重构能力的最小单元。一个 N 输入 LUT 本质上是一个 2N×1 的 SRAM，地址线接输入信号，存储内容即为真值表输出。换言之，一个 4 输入 LUT 可以实现任意 4 变量布尔函数，总共有 22N = 216 = 65536 种不同的函数。

LUT 的妙处在于「以存储换逻辑」：不再用固定门电路实现特定函数，而是通过改写 SRAM 内容来重新定义函数行为。这也是 FPGA「现场可编程」的物理基础。除组合逻辑外，LUT 还可配置为小型分布式 RAM、FIFO 或移位寄存器，进一步提升面积利用率。

{{< docpage "pdf/computer-system/L06-FPGA-and-CGRA.pdf" page=7 >}}

2.2 CLB：逻辑单元的「积木」

可配置逻辑块（Configurable Logic Block，CLB）是 FPGA 架构中的基本组织单元。以 Xilinx 7 系列为例，每个 CLB 包含两个 Slice，每个 Slice 又包含 4 个 LUT、多个触发器（Flip-Flop，FF）以及丰富的多路选择器（MUX）。

触发器提供时钟边沿触发的寄存能力，使 CLB 能够同时实现组合逻辑和时序逻辑。多路选择器则负责将 LUT 输出、触发器输出以及进位链信号灵活地路由到 CLB 内部或外部。进位链（Carry Chain）是 FPGA 中实现高速算术运算（如加法器、计数器）的关键专用资源，它将相邻 LUT 级联，避免通用互连带来的延迟惩罚。

2.3 互连资源与 IOB

如果说 LUT 和 CLB 是「计算节点」，可编程互连网络就是 FPGA 的「高速公路系统」。现代 FPGA 采用分层互连架构：本地连线连接相邻 CLB，长度适中的线段覆盖中等距离，长线则横跨整个芯片用于全局信号（如时钟、复位）分发。

可编程互连点（PIP）和开关矩阵（Switch Box）构成了互连的可重构性：通过配置比特控制晶体管开关的通断，即可在芯片上「绘制」出任意的电路拓扑。IO 块（IOB）则是芯片与外部世界的接口，支持多种 I/O 标准（LVCMOS、LVDS、SSTL 等），可配置驱动强度、摆率和上拉/下拉电阻。

三、配置比特流：FPGA「编程」的真相

很多初学者会疑惑：FPGA 编程和写 C 代码到底有什么区别？答案是根本性的差异。

写 C 代码最终生成的是一串在 CPU 上顺序执行的指令；而 FPGA 的「编程」实际上是在生成一帧配置比特流（Configuration Bitstream）。这帧比特流中每一位都有明确的物理意义：某一位为 1，可能意味着打开某个开关矩阵的特定通路；另一位为 0，可能意味着将某个 LUT 的 SRAM 单元置为低电平。

上电或复位后，FPGA 的配置控制器将这帧比特流逐位写入配置存储器（通常也是 SRAM 阵列）。一旦配置完成，芯片内部的逻辑功能和互连拓扑即被冻结，电路以纯硬件方式并行运行，不再有任何「软件执行」的概念。这种「以空间换时间」的策略，使得 FPGA 在执行数据流规整的算法时，可以达到接近 ASIC 的能效，同时保留重新配置的灵活性。

需要强调的是，FPGA 的配置存储器基于易失性 SRAM，掉电后配置丢失。因此商用 FPGA 通常外挂 Flash 或 EEPROM，上电时自动加载比特流。部分新型 FPGA 也支持部分重配置（Partial Reconfiguration），允许在运行中动态更新芯片的某个区域，其余区域不受影响，这对多任务加速场景极具价值。

四、时序约束：建立时间、保持时间与关键路径

FPGA 虽然摆脱了指令开销，却引入了另一层面的复杂性：时序。数字电路要正确工作，信号必须在时钟边沿到来之前稳定下来，这就引出了两项基本约束。

4.1 建立时间与保持时间

建立时间（Setup Time, Tsu）：数据信号必须在时钟上升沿到来之前至少稳定 Tsu 的时间。若数据到达过晚，触发器可能采样到亚稳态或错误值。

保持时间（Hold Time, Th）：时钟边沿到来后，数据信号必须继续保持稳定至少 Th 的时间。若数据变化过快，同样会导致采样错误。

这两项参数由器件工艺决定，与 FPGA 型号直接相关。例如 Xilinx Kintex-7 的触发器建立时间约为 0.04 ns，保持时间约为 0.2 ns。设计者的目标是通过合理的布局布线，确保所有路径都满足这两项约束。

4.2 关键路径与时序收敛

在同步数字电路中，信号从源寄存器出发，经过组合逻辑和互连延迟，最终到达目的寄存器。整个路径的总延迟必须小于时钟周期，同时满足建立时间和保持时间的要求。其中延迟最长的那条路径被称为关键路径（Critical Path），它直接决定了电路能运行的最高时钟频率。

FPGA 开发流程中的「时序约束」就是向综合与布局布线工具声明时钟频率、I/O 延迟、多周期路径等信息，工具会据此优化逻辑映射和物理布局。若最差路径的 Slack（裕量）为正，则称设计达到「时序收敛（Timing Closure）」；若为负，则需要通过流水线插入、逻辑重构或降低时钟频率来解决。

五、FPGA vs ASIC vs CPU/GPU：架构光谱上的定位

理解 FPGA 的最佳方式，是把它放在架构光谱的恰当位置上。

FPGA 的甜点区在于「算法相对固定、但还需要一定灵活性」的场景。例如 5G 基带处理需要支持不断演进的协议标准，ASIC 流片赶不上标准迭代；CPU/GPU 能效又不足以支撑基站功耗预算。FPGA 正好卡在中间，既能通过重新配置支持新标准，又能提供比通用处理器高一个数量级的能效。

六、CGRA：粗粒度可重构阵列

FPGA 的细粒度可重构带来了极高的灵活性，但也付出了代价：每个 LUT 只有 4~6 个输入，实现一个 32 位加法器需要消耗数十个 LUT 和大量的互连资源，且互连延迟随路径长度线性增长。当应用场景聚焦于字级数据通路（如 16/32 位整数或浮点运算）时，这种比特级可重构显得过度且低效。

CGRA（Coarse-Grained Reconfigurable Array，粗粒度可重构阵列）应运而生。与 FPGA 的 LUT/FF 不同，CGRA 的基本计算单元是 ALU、乘法器、寄存器文件等「粗粒度」功能单元，数据宽度通常为 16 位或 32 位。这些处理单元（PE）以二维阵列形式排布，通过可编程的局部互连网络连接。

6.1 架构特征

CGRA 的典型架构包含：阵列化的 PE、配置存储器、配置控制器，以及可选的数据存储器（Register File 或 Scratchpad）。每个 PE 内部包含一个 ALU、几个本地寄存器、以及连接到邻域 PE 的路由端口。由于互连局限于局部邻域，布线延迟短且可预测，时钟频率通常远高于 FPGA。

配置方面，CGRA 与 FPGA 类似，也是通过配置字定义每个 PE 的功能和互连状态。但由于粒度较粗，配置数据量远小于 FPGA，重配置时间更短，部分 CGRA 甚至支持周期级动态重配置。

6.2 数据流执行模型

CGRA 的执行模型可以从两个维度刻画：调度方式（静态/动态）和执行方式（顺序/数据流）。

静态调度意味着每个 PE 在特定时间段内绑定单一操作，由编译器在编译期完成操作到资源的映射；动态调度则允许运行时根据数据可用性动态分配操作。数据流执行中，操作在操作数就绪后立即触发，无需全局程序计数器驱动。

常见的组合包括：静态调度顺序执行（最简单，配置一次后按节拍推进）、静态数据流执行（编译期确定数据流图映射，运行时按令牌触发）、动态数据流执行（运行时动态匹配操作数令牌，灵活性最高但硬件复杂）。对 AI 加速这类数据流规整的应用，静态数据流模型最为流行：编译器将计算图映射到 PE 阵列，运行时只需不断泵入数据，PE 间以生产者-消费者方式传递结果，天然避免了访存瓶颈。

七、AI 加速器中的应用案例

可重构架构在 AI 推理加速领域找到了最肥沃的土壤。深度学习推理具有鲜明的特征：计算图静态、数据复用模式规律、对延迟和能效敏感。这些特征与 FPGA/CGRA 的能力高度契合。

7.1 FPGA 在数据中心推理

微软在其 Project Brainwave 中将 FPGA 部署到 Azure 数据中心，直接在 Stratix 10 FPGA 上运行已训练的 DNN 模型。通过将矩阵乘法映射到 FPGA 的 DSP 块阵列、激活函数映射到 LUT 实现的查找表、层间数据流通过片上 BRAM 缓冲，Brainwave 在批量大小为 1 的实时推理场景下实现了比 GPU 更低的延迟。

另一类方案如 Xilinx Vitis AI 和 Mipsology Zebra，允许开发者在「零硬件设计」的前提下，将 GPU 训练的模型直接部署到 FPGA。底层工具链自动完成模型量化、算子到 FPGA 原语的映射、以及数据流流水线的生成，极大降低了使用门槛。

7.2 CGRA 在边缘端推理

边缘侧 AI 对能效的要求更为苛刻。CGRA 由于互连简单、控制逻辑轻量，在相同工艺下通常比 FPGA 能效更高。例如三星的 Neural Processing Unit（NPU）和多家学术原型（如 ADRES、Plasticine）都采用 CGRA 架构：编译器将卷积层的循环嵌套映射为 PE 阵列上的静态数据流，权重和激活值通过片上网络广播或轮转传递，几乎无需访问外部 DRAM。

CGRA 的优势在稀疏神经网络和动态图场景下更为突出。部分新型 CGRA 支持「空域-时域」联合映射：在空间维度上并行展开多个输出通道，在时间维度上通过重配置切换不同层的操作类型，实现单一阵列对完整网络的逐层加速。

八、课后思考

  1. LUT 本质上是一个 SRAM。假设一个 6 输入 LUT 采用 64×1 的 SRAM 结构，若 FPGA 芯片上共有 10 万个 LUT，仅 LUT 的 SRAM 就需要多少存储容量？这解释了为什么 FPGA 的静态功耗通常远高于 ASIC 吗？
  2. FPGA 的「配置比特流」与 CPU 的「机器码」在执行机制上有何本质区别？为什么 FPGA 配置完成后不再消耗「取指能耗」，却需要关注「时序收敛」？
  3. CGRA 选择粗粒度 ALU 而非细粒度 LUT 作为基本单元，在灵活性、能效、编译复杂度三方面分别带来了哪些取舍？你认为自动驾驶中的 CNN 推理更适合 FPGA 还是 CGRA，为什么？
  4. 建立时间和保持时间的约束在 FPGA 和 ASIC 中都需要考虑，但 FPGA 通常更难达到高频。结合互连资源的差异，分析背后的根本原因。
