引言
什么是 Vision-Language Pre-training？

Vision-Language Pre-training（VLP）是指通过大规模图文配对数据，预训练能够同时理解视觉和语言两种模态的基础模型。核心目标是学习跨模态的统一表征空间，使模型能够执行图文检索、VQA、图像描述、视觉推理等多种任务。

2021 年 CLIP 的出现标志着一个新的起点：用对比学习将图像和文本映射到同一语义空间。此后，BLIP 解决了"同时理解和生成"的问题，Q-Former 解决了"如何用冻结的大模型高效桥接"的问题，LLaVA 将指令微调引入了多模态领域。每一步都在解决上一步留下的结构性瓶颈。

范式一
对比学习：CLIP → ALIGN → SigLIP

对比学习是 VLP 最基础的训练信号。核心思想：让匹配的图像-文本对在嵌入空间中更接近，不匹配的对更远离。

CLIP (OpenAI, 2021)

CLIP 使用双塔架构（ViT 图像编码器 + Transformer 文本编码器），在 4 亿网络爬取的图文对上做 softmax 对比损失训练。给定批次中的 N 个图文对：

图像到文本损失：

\(\mathcal{L}_{i2t} = -\frac{1}{N} \sum_{i=1}^N \log \frac{\exp(S_{ii}/\tau)}{\sum_{j=1}^N \exp(S_{ij}/\tau)}\)

CLIP 的核心贡献是证明了对比目标 + 海量噪声数据足以学到强大的视觉-语言表征，并具备出色的零样本迁移能力。但局限也很明显：只有编码能力，没有生成能力。

SigLIP (Google DeepMind, 2023)

SigLIP 用 sigmoid pairwise loss 替代 softmax-based InfoNCE 损失：

\(\mathcal{L} = -\frac{1}{n} \sum_{i=1}^{n} \log \frac{1}{1 + e^{z_{ij}(-t) + b}}\)

Sigmoid loss 不需要 batch 内全局归一化，内存开销与 batch size 线性。在 WebLI（10B 图文对）上训练，SigLIP 证明了简单的损失函数改变可以带来显著的效率提升，解耦了 batch size 与损失函数的耦合关系。

演进逻辑：CLIP 证明可行性 → ALIGN 证明数据规模补偿质量 → SigLIP 证明损失函数本身可以更高效。
范式二
统一理解+生成：BLIP

CLIP 只能"编码"，BLIP 让一个模型同时擅长理解和生成。

MED：三种模式共享参数

BLIP 提出 Multimodal Mixture of Encoder-Decoder (MED)，一个模型可切换为三种功能：

  1. Unimodal Encoder：标准编码器，用 ITC 损失对齐视觉-语言表征
  2. Image-Grounded Text Encoder：插入 cross-attention 层注入视觉信息，用 ITM 损失做图文匹配
  3. Image-Grounded Text Decoder：因果 self-attention 做自回归生成，用 LM 损失

三种模式共享 embedding、cross-attention 和 FFN 参数，仅 self-attention 层分叉（编码用双向，解码用因果）。

CapFilt：数据自举

BLIP 引入 Captioner + Filter 机制从噪声 web 数据中自举高质量训练集：

  • Captioner：用 LM 微调的 decoder，为 web 图像生成合成 caption
  • Filter：用 ITC+ITM 微调的 encoder，过滤噪声 caption
  • 两者独立训练（共享参数会导致 confirmation bias）

在相同 14M 数据量下，BLIP 比 ALBEF 在 COCO 检索上提升 +2.7% 平均 Recall@1，VQA 上提升 +1.6%。

范式三
高效桥接：BLIP-2 / Q-Former

BLIP-2 解决了一个全新的问题：如何用冻结的视觉编码器和冻结的 LLM 做 VLP？

Q-Former：信息瓶颈

Q-Former 是一个 188M 参数的轻量桥接模块，使用 32 个可学习 query embeddings 从冻结 ViT 中提取特征。输出仅为 32×768（远小于 ViT-L 的 257×1024），形成信息瓶颈，迫使 queries 只提取与文本最相关的视觉信息。

两阶段训练

Stage 1: Vision-Language Representation Learning — 连接冻结 ViT，联合优化 ITC + ITG + ITM：

  • ITC：unimodal mask，query 和 text 互不可见
  • ITG：causal mask，训练 query 提取生成所需的视觉信息
  • ITM：bidirectional mask，细粒度图文匹配

Stage 2: Generative Learning — 连接冻结 LLM（OPT 或 FlanT5），Q-Former 输出经 FC 层投影为 soft visual prompts 前置于 LLM 输入。

消融实验证明：跳过 Stage 1 会导致 OPT 严重的灾难性遗忘。两阶段训练不可或缺。

惊人效率

BLIP-2 在 zero-shot VQAv2 上达到 65.0% 准确率，比 Flamingo-80B 的 56.3% 高出 8.7%，但可训练参数仅 108M，是 Flamingo 的 1/54。

可扩展性：更强的视觉编码器或更强的 LLM 都持续提升性能，验证了 BLIP-2 作为通用框架的灵活性。
范式四
Few-shot 学习：Flamingo

Flamingo（DeepMind, 2022）走了另一条路：冻结 LLM + 视觉注入 + few-shot prompting。

三大架构创新

  1. Perceiver Resampler：将可变大小的视觉特征压缩为 64 个固定 token
  2. Gated XATTN-DENSE：在冻结 LM 层间插入 cross-attention 层，输出乘以 tanh(α)，α 初始化为 0，确保训练初期等价于原始 LLM
  3. Image-causal masking：每个 text token 只 cross-attend 到紧邻其前的图像，通过 LM self-attention 间接依赖之前所有图像

交错数据训练

Flamingo 在 M3W 数据集（43M 网页的交错图文）上训练，这是 few-shot 能力的关键。加上 ALIGN（1.8B）、LTIP（312M）和 VTP（27M），Flamingo 在 6/16 任务上 few-shot 性能超越了 fine-tuned SOTA。

Flamingo 的 Perceiver Resampler 和 Gated XAttn 直接影响了 BLIP-2 的 Q-Former 设计。但 Q-Former 更轻量（32 vs 64 queries）且用两阶段训练替代单阶段，效率更高。

范式五
指令微调：LLaVA

LLaVA（UW/Microsoft, 2023）将 NLP 的 instruction tuning 首次系统性扩展到多模态空间。

极简架构

CLIP ViT-L/14（冻结）→ 简单线性层（可训练）→ Vicuna LLM（可训练）。没有 Q-Former、没有 Perceiver，就是一个 MLP。

GPT-4 生成训练数据

LLaVA 用语言-only GPT-4 将 COCO 的 image-text pairs 转化为 158K 多模态指令跟随数据，包含三类：Conversation (58K)、Detail Description (23K)、Complex Reasoning (77K)。

反直觉发现：简单 MLP 连接器在很多任务上表现优异，说明连接器复杂度不是瓶颈——数据质量和 LLM 能力更关键。LLaVA 确立了 "CLIP encoder + MLP projector + LLM" 的 MLLM 基础架构范式。

补充
细粒度对齐：GLIP

GLIP（Microsoft, 2022）从另一个维度推进 VLP：从图像级对齐到对象级。

核心洞察：目标检测 = 上下文无关的短语定位。GLIP 将检测器的分类头替换为 region-word 对齐得分（S = O · PT），统一了检测和定位任务。通过 deep fusion（图像-语言编码器间的交叉注意力）使视觉表征具有语言感知能力。

GLIP 用 self-training 为 24M web 图文对自动生成 grounding box（78.1M 伪标注），实现了 zero-shot COCO 检测 49.8 AP（超越监督 Faster R-CNN ResNet-101 的 42.0）。

演进脉络
时间线与发展趋势

2021：对比学习范式确立。CLIP 证明大规模对比学习可行，ALIGN 证明数据规模补偿质量。

2022：统一架构 + Few-shot + 细粒度对齐。GLIP 从图像级推进到对象级；Flamingo 证明冻结 LLM 的 few-shot 学习；BLIP 统一理解与生成。

2023：高效桥接 + 指令微调 + 训练优化。BLIP-2 的 Q-Former 确立冻结大模型 + 轻量桥接范式；LLaVA 引入视觉指令微调；SigLIP 优化训练效率。

2024-2026：开源追平商业 + 统一理解生成。InternVL、Cambrian-1 等开源模型达到 GPT-4V 级性能；BAGEL 等工作探索统一 tokenizer 的理解+生成；多模态 Agent 能力和视频理解成为新前沿。

一句话总结：CLIP 解决了"能不能对齐"，BLIP 解决了"能不能同时理解和生成"，Q-Former 解决了"能不能用现成大模型高效桥接"，LLaVA 解决了"能不能像人一样对话式交互"。
展望
开放问题与未来方向
  1. 连接器的最优设计：Q-Former vs MLP vs Perceiver？复杂连接器是否必要？
  2. 视觉编码器的 scaling：ViT 参数远小于 LLM，视觉侧是否需要 scaling up？
  3. 统一理解+生成：理解和生成是否需要统一的 tokenizer/架构？
  4. 数据工程上限：CapFilt/self-training/合成数据的天花板在哪？
  5. 视频理解：如何高效建模时序信息？帧级编码是否足够？
  6. 多模态 Agent：VLM 如何与工具使用、规划、记忆等 Agent 能力融合？
